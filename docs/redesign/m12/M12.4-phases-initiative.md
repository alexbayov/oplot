## TL;DR

Все мобы (не только боссы) получают **фазу 2** при 50% HP. Initiative order становится **видимым на 5 ходов вперёд**.

После M12.4 каждый бой имеет внутреннюю динамику (моб на 50% меняет поведение → игроку нужно адаптироваться), а игрок планирует ходы вперёд (initiative timeline).

**Длительность:** 2 недели, 2 PR.

---

## 1. Mob Phase 2

Каждый из 11 мобов получает trigger «при HP < 50% → новое поведение / способность»:

| Mob | Phase 1 | Phase 2 (при 50% HP) |
|---|---|---|
| marauder | стандартный | seekCover behaviour — прячется, accuracy +20% |
| wild_dog | rush attack | frenzy — damage +30%, статус bleed always |
| mutant | melee | roar AOE fear (telegraph за 1 ход) |
| looter_sniper | sniper telegraph | repositions, telegraph crit shot ×4 |
| armored_guard | tank | armor breaks: -defense -50%, +20% damage |
| fanatic_berserker | уже умеет | максимальный frenzy + immune to fear |
| pack_rat | стая | summon +1 rat каждые 2 хода |
| relic_drone | ranged | overdrive: -50% damage taken, +50% damage out, 2 хода |
| forest_alpha_mutant (boss) | mid attacks | alpha rage: fear AOE + heal себе 30 HP |
| warehouse_drone_prime (boss) | shield phase | shield down: vulnerable -50% but +50% damage |
| city_guard_captain (boss) | leader buff | rallying cry: AOE buff союзникам +30% damage |

### Реализация

`mob.phase2_ability`, `mob.phase2_behavior_id` в `mobs.json`. CombatEngine отслеживает HP, при пересечении 50% → событие `mobPhaseChange` → применяет новое behaviour и опционально мгновенное ability (rage).

Визуальный эффект: вспышка над мобом + sound + текст-флоат «ярость!».

## 2. Initiative Timeline

Под action bar — горизонтальная **timeline** с 5 предстоящими ходами:

```
[hero]  [marauder]  [hero]  [dog]  [mutant]  [hero]  ...
 ↑                                                    ↑
сейчас                                       5 ходов вперёд
```

### Что показывает
- Кто ходит и в каком порядке (initiative-based)
- Икона actor (портрет 24px) + indicator (телеграф если есть)
- При наведении — tooltip «На этом ходу sniper telegraph будет execute»
- Текущий ход — золотая обводка

### Что меняет timeline
- Stim (+1 action) → дополнительный hero slot
- Stun → пропуск slot моба
- Smoke (escape) → таймлайн обнуляется

## 3. State Integration

```ts
interface CombatState {
  // ...
  initiativeOrder: string[];           // actor ids, next 5+
  upcomingActions: PlannedAction[];    // M12.2 telegraphs + M12.4 phases
}

interface PlannedAction {
  actorId: string;
  turnsAhead: number;
  abilityId?: string;   // telegraph or signature
  icon?: string;
}
```

CombatEngine.recomputeTimeline(): обновляется после каждого хода с учётом cooldown'ов, статусов, фаз.

## 4. Balance

GD пишет отдельный документ `docs/balance.md` секция «M12.4 phase transitions»:
- Фаза 2 должна **менять тактику** игрока, не просто «больше damage»
- Эталонные бои: для каждого моба — 2 сценария (фаза 1 и фаза 2 проявления)
- Игрок должен иметь способ среагировать (если frenzy → лечись или беги; если armor breaks → переключи оружие)

## 6. Открытые вопросы

1. **Фаза 2 «отменяется» при heal обратно > 50% HP?** Нет, one-way trigger (иначе можно зацикливать).
2. **Phase 2 для всех мобов или скиповать слабых?** Для всех 11 — иначе игрок не учится паттерну.
3. **Timeline показывает только героя и mobs или env-events?** Только акторов; env-events если они triggered (например, бочка с таймером).
4. **Phase 3 для боссов?** Out of scope M12.4. Возможно в M13+.
5. **Видимость phase в HP-баре?** Phase 1 — green; phase 2 — red bar или другой цвет.
