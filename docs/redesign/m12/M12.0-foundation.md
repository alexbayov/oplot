## TL;DR

Делаем бой готовым к большим изменениям, **не меняя сам бой**. После M12.0 игра играется как сейчас, но:
- Под капотом — новый `CombatEngine` (фасад) и `CombatState` (state machine)
- M11 поля (calibre, magazineSize, mods, durability) **доходят до боя**
- Игрок выбирает цель кликом, видит HP-бары, видит свой магазин, видит tooltip оружия
- Save migration v2 → v3 (cooldowns, statuses, position готовы в shape, но пустые)

**Длительность:** 2 недели, 3 PR.

## 3. M11 поля доходят до calc

Сейчас `applyAttack(hero, mob, weapon, rng)` использует только `weapon.damage_min/max`. Меняем:

### 3.1 Magazine + reload
- `actor.magazine` инициализируется из `equipped.magazineSize` (или null для мили)
- Каждая атака из ranged → `magazine -= 1`
- Если `magazine === 0` → action button "ПЕРЕЗАРЯДКА" (1 ход) появляется автоматически
- В backpack тратятся ammo соответствующего calibre

### 3.2 Mod effects
- При сборке `WeaponInstance` агрегатор считает effective stats:
  - `damageMin += sum(mod.effects.damageDelta) for installed mods`
  - `noise = mod.effects.noiseSet ?? weapon.noise`
  - `accuracy += sum(mod.effects.accuracyDelta)`
  - `magazineSize += sum(mod.effects.magazineDelta)`
- `combat.calcDamage` использует уже агрегированные значения

### 3.3 Durability
- При попадании по мобу → `weapon.durability -= 1`
- При промахе → `-= 0.5` (округляется)
- При `durability === 0`:
  - Самодел → `breaksInto: broken_craft_*` (по решению Alex)
  - Дроп → instance уничтожается, возвращаются 30% parts (durability.ts уже умеет)

## 4. UI изменения

### 4.1 HP-бары
Под портретом каждого моба — `<div bar 0..100%>` вместо числа. Цвет: green > 60%, yellow 30-60%, red < 30%.

Hero HP — тоже бар, плюс число рядом (для precision).

### 4.2 Targeting
- При клике на портрет моба → `state.target = mob.id`
- Активный target — золотая обводка
- При нажатии "АТАКА" / "ПРИЁМ" → бьётся `state.target`
- Если target killed → автовыбор следующего живого

### 4.3 Magazine indicator
Рядом с кнопкой "АТАКА" — `[7/8]` (текущий магазин / max). Красная подсветка если ≤2.

### 4.4 Weapon tooltip
Long-press / hover на иконке оружия в action bar:
```
АК-74 [T4]
Калибр: 5.45×39
Урон: 8-12 + crit 5%
Магазин: 30 (+ ПБС не установлен)
Шум: высокий
Прочность: 67/100
[ПБС] [Прицел]
```

## 7. Открытые вопросы

1. **Перезарядка стоит хода или free action?** Рекомендация: 1 ход. Иначе магазин не имеет тактического значения.
2. **Сколько модов одновременно на оружии?** По типу: каждый ствол может иметь 0-3 modSlot (ствол/прицел/магазин). См. M11.0-weapons.md.
3. **Тactile feedback при низком магазине?** Yes — красная подсветка кнопки АТАКА если magazine ≤ 2.
4. **Durability — показывать игроку или скрыть?** Показывать в tooltip как «прочность 67/100», в action bar — иконка только если ≤30%.
