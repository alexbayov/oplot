# M10.2 — Encounter Taxonomy

> 30+ encounter'ов без структуры превратятся в «обыскать / обойти». Эта таксономия предотвращает.
> Версия: v1 · Автор: Zo · PM: Alex

---

## Зачем таксономия

В M10.2 предполагается 30–50 encounter'ов. Без чёткой структуры рисков три:

1. **Монотонность** — 70% событий сведутся к «риск vs награда» (обыскать → возможно лут / возможно урон). Игрок через 5 минут начинает скипать.
2. **Невозможно генерить пулом** — без шаблонов LLM выдаст generic «древний рюкзак / странный звук». Не будет postsov-атмосферы Оплота.
3. **Невозможно балансить** — если каждый encounter уникальная снежинка, нельзя сказать «у нас слишком много trap'ов и мало moral choice».

Решение: **6 категорий**, каждый encounter принадлежит одной (плюс опциональный тэг). Pool балансируется: ~30% самый частый тип, ~5% самый редкий.

---

## 6 категорий

### 1. **Resource Trade** (≈25% pool) — обмен с риском

Игрок жертвует X, получает шанс на Y. Прозрачная математика.

**Outcome форма:** `cost (HP/ресурс/время) → loot (вероятностный)`

**Примеры:**
- «Ржавый сейф под завалом. Чтобы достать — час долбить ломом. Время = +1 ход до возврата.»
  - 80% +3 scrap, +1 cloth · 20% сломанный лом (-1 cond) · cost: +1 ход
- «Лужа с маслянистой жидкостью. Можно набрать в флягу, но воняет химией.»
  - 100% +2 chem, но 30% -5 HP (отравление)

### 2. **Moral Choice** (≈15%) — выбор без правильного ответа

Два варианта, оба что-то стоят. Нет «оптимального». Тон важнее награды.

**Outcome форма:** `choice A → A_consequence; choice B → B_consequence` (обе ощутимые)

**Примеры:**
- «У стены сидит человек. Хрипит. Дать аптечку — он встанет и уйдёт. Не дать — может, потом найдёшь его лут.»
  - дать: -1 medkit, +1 «доверие» (мета-флаг, влияет на radio/encounter pool)
  - не дать: 60% +1 medkit на трупе на возврате · 40% человек ушёл, ничего

- «Дети у костра. Делят сухарь. Можно поделиться запасами — все будут сыты, или пройти мимо.»
  - поделиться: -2 food, +1 «доверие»
  - пройти: ничего, но в дневнике запись

### 3. **Trap / Combat Preview** (≈15%) — ловушка или подсказка о бое

Игрок что-то задевает, получает урон ИЛИ предупреждение о моба впереди.

**Outcome форма:** `instant cost OR forewarning`

**Примеры:**
- «Растяжка от консервной банки. Звон.»
  - 100% следующий моб начнёт бой с инициативой (теряем 1 ход)
- «Запах гари. Дальше явно был костёр недавно.»
  - 100% следующий мобу +20% HP (был на отдыхе), но игрок может развернуться (-1 depth-credit)

### 4. **Lore Beat** (≈15%) — текст без gameplay-выбора

Атмосфера, мир, прошлое. Никакого выбора. Просто 2-3 предложения, иногда +1 lore-collectible.

**Outcome форма:** `narrative only, optional +1 lore_fragment`

**Примеры:**
- «На стене мелом: "Они приходят с дождём. Не верь радио."»
  - +1 lore_fragment (в дневнике)
- «Старая детская обувь у двери. Одна. Кто-то очень торопился.»
  - just flavor

Lore-beats повторяются реже (rarity tier).

### 5. **Skill Check** (≈15%) — проверка перка или экипировки

Условный gate. Если есть X — открывается путь. Если нет — заглушка.

**Outcome форма:** `if has(X) → bonus path; else → default`

**Примеры:**
- «Замок на ящике. Нужен взлом-инструмент.»
  - если есть `lockpick` в инвентаре: 100% +5 scrap, +1 ammo
  - иначе: пройти мимо (заглушка)
- «Глубокая лужа. Чтобы пройти — броня будет мокрая (-1 cond), либо обходить (+1 ход).»
  - перк `Outdoorsman`: можно пройти без потерь
  - иначе: выбор cond -1 vs ход +1

### 6. **NPC Cameo** (≈15%) — короткая встреча с фигурой

Не quest-giver, не торговец — просто **присутствие другого человека**. Создаёт ощущение «не один в пустоши». Иногда +info, иногда +trade, иногда просто диалог.

**Outcome форма:** `dialog → optional trade / info / quest_seed`

**Примеры:**
- «У бочки греется старик. "Ты тоже в Завод? Не ходи. Там собаки." Кивает на восток.»
  - 100% raw info (помечает forest_alpha_mutant в encounter pool +20% дроп)
- «Бродяга. Хочет обменять флягу воды на патрон.»
  - принять: -1 ammo, +1 water
  - отказать: ничего, но он бросает в спину: «Ну и подыхай.»

## Правила пула

1. **Зональная фильтрация.** Каждый encounter имеет `zones: ["forest", "warehouse", ...]` или `zones: ["*"]` (универсальный).
2. **Cooldown.** Один и тот же encounter не повторяется в течение 5 вылазок.
3. **Распределение в одной вылазке.** Если вылазка содержит 3 encounter'а — они должны быть из **разных** категорий. Без двух trap'ов подряд.
4. **Lore rarity.** Lore-beat имеет 0.5× weight против остальных (он наградой воспринимается, не должен надоедать).
5. **Skill check fallback.** Если у игрока нет нужного перка/предмета, encounter должен иметь читабельный «пройти мимо» исход. Не softlock.

## JSON schema (предлагается)

```json
{
  "id": "rusty_safe",
  "category": "resource_trade",
  "zones": ["warehouse", "factory"],
  "text_ru": "Ржавый сейф под завалом...",
  "illustration": null,
  "choices": [
    {
      "id": "force_open",
      "text_ru": "Долбить ломом",
      "requires": [],
      "outcomes": [
        { "weight": 0.8, "effect": { "loot": [{"id": "scrap", "n": 3}], "time": 1 } },
        { "weight": 0.2, "effect": { "damage_item": "lockpick" } }
      ]
    },
    { "id": "skip", "text_ru": "Не лезть", "outcomes": [{ "weight": 1, "effect": {} }] }
  ]
}
```

`requires` поддерживает: `has_item: <id>`, `has_perk: <id>`, `min_hp_pct: 0.5`, etc.

## Pipeline разработки контента

1. **Первые 6** — пишутся вручную, по одному на категорию. Это «эталоны тона».
2. **Эталоны → промт для GPT** — даём LLM 6 эталонов, просим 30 новых в их духе.
3. **Каждый сгенерированный — ручная правка** (тон, postsov, отсутствие фэнтези-штампов, лор Оплота).
4. **Sanity-pass** — есть ли readable «пройти мимо», работает ли requires-чек, не дублируется ли с существующим.
5. **Балансировочный pass** — сошлось ли распределение по категориям?

Длительность контента: 4-5 часов после написания эталонов.

## Acceptance для M10.2 (обновлённый)

- [ ] Минимум 6 эталонных encounter'ов (по одному на категорию) написаны вручную
- [ ] Минимум 24 дополнительных через pipeline (итого 30 на старте)
- [ ] Распределение по категориям выдерживается ±5%
- [ ] Schema validated через тест
- [ ] Cooldown работает (тест на 5 последовательных вылазок без повтора)
