# M10.3 — Base Wireframe / Hotspot Layout

> Painted interior с 5+ hotspots — это не «нагенерил вариантов и выбрал». Сначала composition, потом генерация.
> Версия: v1 · Автор: Zo · PM: Alex

---

## Проблема

В исходном M10.3 спеке предполагается генерация 3-4 вариантов painted interior через Gemini → выбор лучшего. Но **hotspots должны лежать в композиционно работающих местах**, а не там, где LLM их нарисовал. Если сначала генерировать — потом окажется что верстак идеально нарисован в углу, который заслонён лежанкой, или радио на стене, которую видно только на 20% кадра.

Решение: **wireframe → utверждение раскладки → painted-генерация под раскладку**.

## 6 hotspots (приоритезированы)

| # | Hotspot | Что заменяет | Действие | Painted элемент | Состояния |
|---|---|---|---|---|---|
| 1 | **Hero portrait + HP/XP** | левая колонка сейчас | tap → детали perks/stats | Герой сидит на ящике / лежанке, костёр перед ним | normal · HP<50%: бинт · HP<20%: кровь на одежде |
| 2 | **Workbench** | Мастерская tab | tap → CraftScene overlay | Верстак с деталями, заготовка винтовки | normal · «in progress» (видна заготовка от последнего крафта) |
| 3 | **Stash chest** | Инвентарь tab | tap → InventoryScene overlay | Деревянный ящик, открытая крышка, видно слои тряпок | normal · «overweight» иконка если >80% веса |
| 4 | **Radio** | Радио tab | tap → RadioScene overlay | Старая радиостанция на полке, антенна | normal · мигает красным если unread signal · полупрозрачные волны |
| 5 | **Kettle / Fire** | Прогрессия/Перки tab | tap → ProgressionScene overlay | Костёр + чайник, пар поднимается | always animated (loop) — это «душа» сцены |
| 6 | **CTA button** | «В ВЫЛАЗКУ» | tap → MapScene | Painted кнопка-табличка над дверью наружу | normal · pulse glow |

**Day counter** — рендерится поверх top bar (top-left). Не hotspot, но визуально якорит «время идёт».

## Динамические слои (поверх painted background)

### Always-on
- `kettle_steam.png` (256×128) — pixel-displaced loop, очень субтильный (~0.4 opacity)
- `lamp_flicker` — Phaser tween на opacity лампы (0.85 ↔ 1.0, 3.2s loop)

### Conditional
- `radio_blink` — красный диод 8×8 пикселей, мигает 0.8/0.4s, только если `activeSignals().some(s => !s.read)`
- `window_lighting_overlay` — 4 варианта (day/dusk/night/dust). Применяется по `GameState.day % 4` (или явный weather flag)
- `hero_state_overlay` — поверх hero portrait: bandage sprite если HP<50%, blood splat если <20%
- `workbench_inprogress` — visible если есть `recentCraft` в последние N часов

### Один-два «just for life»
- Силуэт кошки на лежанке (50% шанс при загрузке сцены — иногда есть, иногда ушла гулять)
- Капля с потолка (изредка), tween на y и opacity → splash

## Композиция: правила для генерации

Это инструкции для генератора (Gemini / художника):

1. **Side-view, не isometric.** Как Banner Saga base camp, не Fallout Shelter.
2. **Light source = костёр в центре-низу.** Тёплый оранж. Тени от центра — холодные серые / синие.
3. **Background depth.** 3 плана: пол + стены (фон), мебель (mid), герой + объекты-hotspots (передний). Painted graphic novel — лёгкий outline на переднем плане, плавно к фону.
4. **Цветовая палитра.** Совпадает с `style-guide.md`: гр­язно-серый, землистый, акценты тёплого золотого (#C5A267) на металле/огне.
5. **Композиционные акценты — на hotspots.** Workbench с тёплым local light. Stash с открытой крышкой (читается что внутри что-то есть). Radio на стене ровно на eye-level.
6. **Hero portrait — НЕ строго по центру.** Слева-внизу, на 1/3 ширины. Центр кадра — костёр / kettle.
7. **CTA в нижнем правом.** Painted табличка или дверной проём с надписью / стрелкой наружу.
8. **Никаких очевидных UI-элементов в painted.** Кнопки, иконки, бары — отдельным слоем сверху, динамические.

## Pipeline

1. **Whiteboard sketch** (этот документ) → утверждение раскладки с Alex'ом
2. **Greybox** в Figma / pen on paper — серые прямоугольники на местах hotspots, проверка читаемости композиции
3. **Generation prompt** для Gemini — берётся раскладка + правила выше, генерируется 4-6 вариантов **без вариаций по композиции** (только по освещению / атмосфере)
4. **Top 2** доводятся вручную (PS / Krita) — особенно hotspot edges
5. **Final** экспорт 1280×720 PNG, no alpha (фон — финальный)
6. **Hotspot polygons** — рисуются поверх в коде как `Phaser.Geom.Polygon` (или Rectangle где можно). Hover/tap region для каждого.

## Tooling в коде

- `src/scenes/BaseScene.ts` сильно худеет: становится менеджером hotspots, не layout-логикой
- `src/ui/components/Hotspot.ts` — переиспользуемый компонент:
  ```ts
  hotspot(scene, { x, y, w, h, label, onTap, badge?, glowOnHover: true })
  ```
- Состояния (HP overlay, radio blink, workbench in-progress) — отдельные системы:
  - `src/systems/baseAmbience.ts` — kettle steam, lamp flicker, cat
  - `src/systems/baseState.ts` — derive из GameState какие conditional overlays активны
