## TL;DR

Заменяем «вылазка = 2–4 боя по скрипту» на **управляемый игроком гринд-формат**.

**Игрок входит на Точку → волны мобов появляются с интервалом → бой → лут → пауза → следующая волна → выходит когда захочет.**

Лимиты выхода — **не количество боёв**, а **HP / вес / патроны / голод**. Чем дольше сидишь — тем плотнее и злее волны (механика «уплотнения»).

---

## 1. Концепция

### 1.1 Что такое «Точка»

Точка — это **фиксированный спот внутри зоны**. На карте мира у некоторых зон будет **иконка «Точка»** (костёр / лагерь / стрелочка) — это вход в spot grinding режим.

**Пример:** в зоне «Пригород» есть Точка «Перекрёсток» — конкретная painted локация, на которой сидит игрок и отбивается от волн.

### 1.2 Как это играется

1. Игрок выбирает зону → выбирает Точку → жмёт «Войти на Точку»
2. Переход на новую сцену `SpotScene` (painted background конкретной локации)
3. Через 3–8 игровых секунд — **первая волна** (1–2 моба)
4. Бой в существующем `CombatScene` (без изменений)
5. После боя — лут, возврат в `SpotScene`, пауза
6. Через 5–15 секунд — следующая волна
7. Каждые **N убийств** — шанс на **mini-boss spawn** (жирный моб с гарантированным редким лутом)
8. Игрок жмёт «Вернуться на базу» **между волнами** — выходит со всем лутом

### 1.3 Почему это работает

- **Игрок принимает решение постоянно:** «ещё одну волну или уйти с этим?»
- **Награда видна сразу:** счётчик убийств / лута на HUD
- **Жадность как механика** срабатывает: «остался ещё чуть-чуть, добью одну»
- **Mini-boss как джекпот:** редкое событие, ради которого хочется задержаться

```
        ┌─────────────┐
   ────▶│   IDLE      │  ← начальное состояние при входе

           │ (timer: spawn_delay)
           ▼
        ┌─────────────┐
        │ WAVE_INCOMING│  ← short message "Слышишь шаги..."

        ┌─────────────┐

        └──┬──────────┘
           │ (bой завершён)
           ▼
        ┌─────────────┐
        │   LOOT      │  ← inline loot panel
        └──┬──────────┘
           │ (игрок забирает)
           ▼
        ┌─────────────┐
        │  IDLE       │◀── петля: возвращается в idle, тикает density
        └──┬──────────┘
           │ (игрок: "Вернуться")
           ▼
        ┌─────────────┐
        │  EXIT       │  ← подсчёт сессии, возврат на базу
        └─────────────┘
```

### 2.3 Параметры сессии (хранятся в state SpotScene)

```ts
interface SpotSession {
  spotId: string;
  startedAt: number;
  wavesCompleted: number;
  killsCount: number;
  miniBossesSpawned: number;
  lootCollected: ItemStack[];
  densityLevel: number;    // 1..5, начинается с 1
  nextWaveAt: number;      // timestamp
  state: "idle" | "wave_incoming" | "combat" | "loot" | "exit";
}
```

## 3. Wave System

### 3.1 Генерация волны

Каждая волна = 1–3 моба из пула волны точки. Состав определяется `density_level`.

```ts
// pseudocode
function generateWave(spot: SpotDef, density: number): Mob[] {
  const pool = spot.wavePools.find(p => p.minDensity <= density && density <= p.maxDensity);
  const count = randomInt(pool.countMin, pool.countMax);
  return Array.from({ length: count }, () => pickWeighted(pool.mobs));
}
```

### 3.2 Density progression

`density_level` повышается по **триггерам**:

| Триггер | Δ density |
|---|---|
| Каждые 3 убитых моба | +1 |
| Прошло 2 минуты от старта | +1 |
| Игрок дважды отказался от «вызова боя» (если будет такая опция) | +1 |
| Уход на базу | reset to 1 (при следующем заходе) |

`density_level` cap = 5. На 5 — спавнятся mini-boss-ы чаще + волны более плотные.

### 3.3 Timing

- **Стартовая задержка** до первой волны: **5 секунд** (real time)
- **Между волнами** при density 1: **10 секунд**
- **Между волнами** при density 5: **5 секунд**
- Игрок может **жать «Ждать»** — следующая волна через **1 секунду** (для тех кто фармит на скорость)

## 4. Mini-boss механика

### 4.1 Spawn rules

Mini-boss появляется **вместо** обычной волны со следующими шансами:

| Density level | Mini-boss spawn chance per wave |
|---|---|
| 1 | 0% |
| 2 | 2% |
| 3 | 5% |
| 4 | 10% |
| 5 | 15% |

**Cooldown:** не чаще **1 mini-boss / 5 минут** реального времени (защита от RNG-кластеров).

### 4.2 Поведение mini-boss

- Mini-boss = **отдельный mob_id**, отличный от обычных (`forest_marauder_boss`, `city_thug_boss` и т.п.)
- HP / damage **в 2.5–3× выше** обычного моба той же зоны
- **Гарантированно роняет** 1 редкий дроп (деталь оружия, мод, чертёж)
- Визуально: над портретом — **золотая корона / красная стрелка** в `CombatScene`
- В UI «волна» помечается **алым цветом** заранее: «🩸 Угроза приближается»

### 4.3 Reward масштаб

| Зона | Mini-boss | Гарантированный редкий лут |
|---|---|---|
| Лес | Большой марадёр | Часть Поджиги ИЛИ кожанка |
| Пригород | Бывший милиционер | `pm_slide` ИЛИ `tt_magazine` |
| Город | Главарь банды | `akm_bolt` ИЛИ `mod_pbs_universal` |
| Фабрика | Тяжелобронированный | `ak74_barrel` ИЛИ редкий мод |

## 5. Exit HUD (экстракция)

В правом верхнем углу `SpotScene` — **постоянно видимый HUD** с лимитами выхода.

```
┌────────────────────────┐
│ ❤️ HP:     72/100      │
│ 🎒 Вес:    18/30 кг    │
│ 🔫 9×18:   24 патрона  │
│ 🔫 7.62:   12 патронов │
│ 💀 Убито:  7           │
│ 📦 Лут:    14 предм.   │
│ 🔥 Угроза: ★★☆☆☆      │  ← density visual
├────────────────────────┤
│  [ ВЕРНУТЬСЯ НА БАЗУ ] │  ← active only when state=idle
└────────────────────────┘
```

### 5.1 Лимиты и автовыход

| Условие | Поведение |
|---|---|
| HP ≤ 0 | Death — теряешь часть лута, возврат на базу со штрафом |
| Вес > weight_max + 5 кг | Принудительный выход после следующей волны |
| Патронов = 0 для активного оружия | Предупреждение: «У тебя нет патронов!» — игрок может уйти или переключиться на melee |

### 5.2 Возврат с лутом

- **Обычный выход** (кнопка): весь лут сохраняется
- **Death:** теряется 50% лута (рандомные стеки) + 1 случайный из инвентаря (с шансом 30%)
- **Forced exit по весу:** весь лут сохраняется

## 7. Telemetry (для балансировки)

Точка — **самый легко-сламываемый** формат: тут игрок может уйти в бесконечный гринд или сразу умереть. Нужна телеметрия для пост-релизного тюнинга.

**Логируется на каждый exit/death:**

| Метрика | Цель события |
|---|---|
| `spot_session_started` | spotId, player_level |
| `spot_wave_completed` | spotId, density, kills_total |
| `spot_miniboss_spawned` | spotId, density, kills_total |
| `spot_miniboss_killed` | spotId, hp_remaining_pct |
| `spot_session_ended` | spotId, duration_sec, kills, loot_count, exit_reason: "voluntary" \| "death" \| "weight" |
| `spot_player_died` | spotId, density, kills_before_death |

**Алерты на дашборде:**

- avg(`duration_sec`) > 15 минут → сессии слишком длинные, нужно ужесточить density
- death_rate > 30% → точка слишком сложная
- mini-boss kill rate < 50% при density 5 → mini-boss слишком сильные

## 11. Открытые вопросы

1. **«Ждать» vs автотаймер** — таймер фиксированный или игрок жмёт «следующая волна»? Я бы делал **гибрид**: автотаймер 10-15 секунд **+ кнопка «Не ждать»**. Так и медленные игроки не нервничают, и быстрые не теряют темп.
2. **Painted фоны для точек** — рисуем 4 уникальные локации? Или используем variant существующих zone-backgrounds с overlay (костёр / выжженная земля / труп / баррикады)? Variant дешевле, но беднее.
3. **Mini-boss death — гарантированный лут или просто высокий шанс?** Я бы делал **100% гарантированный 1 редкий + 50% шанс на 2-й**.
4. **Принудительный выход по весу** — мгновенный или со следующей волной? Мгновенный = травматичный, со следующей = риск (можно умереть со всем лутом).
5. **Несколько игроков одновременно на одной точке** (если когда-то будет co-op)? Сейчас single-player, но архитектуру state можно сразу делать с учётом.
6. **Регенерация HP между волнами** — фиксированная (например, +5 HP в IDLE state каждые 10 сек) или только через расходники? Регенерация делает точку **более доступной**, расходники — **более напряжённой**. Я бы делал **0 регенерации**, чтобы расходники имели цену.
