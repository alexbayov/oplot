## Вопрос 1. Реальные названия vs «9мм пистолет»

**Контекст:** Яндекс.Игры теоретически может зарубить контент с явными названиями реального оружия (политика «без оружия и насилия»). Но в постсоветском survival без ПМ/АК — это профанация жанра.

### ✅ Рекомендация: переключатель `WEAPON_NAMING_MODE` в `config.ts`

```ts
// src/config.ts
export const WEAPON_NAMING_MODE: "real" | "generic" = "real";
```

В `content/items.json` для каждого ствола **две пары полей**:

```json
{
  "id": "pm",
  "name_real_ru": "ПМ",
  "name_generic_ru": "9мм пистолет",
  "description_real_ru": "Пистолет Макарова. Калибр 9×18. Простой, надёжный, повсеместный.",
  "description_generic_ru": "Компактный пистолет под слабый патрон. Лёгкий и надёжный."
}
```

UI (`InventoryScene`, `WeaponDetails`) читает поле через хелпер:

```ts
function itemName(item: Item): string {
  return WEAPON_NAMING_MODE === "real" ? item.name_real_ru : item.name_generic_ru;
}
```

**Почему так:**
- Если Яндекс на ревью попросит убрать названия — **одна строка в `config.ts`**, билд пересобирается, контент не трогается
- В разработке/деве всегда играем с реальными названиями (атмосфера сразу правильная)
- В сабмит на YI можно собирать в любом режиме

**Альтернатива:** сразу делать только «9мм пистолет». Минус: теряется характер, ПМ и Стечкин становятся «9мм пистолет №1» и «9мм пистолет №2».

## Вопрос 2. Цена модификации — ресурсы или детали?

**Контекст:** ПБС-1 (глушитель), коллиматор, расширенный магазин, ремень-сошки. Как их крафтить?

### ✅ Рекомендация: **двухуровневая система**

**Базовые моды** (T2–T3) крафтятся **из ресурсов**:

```
ПБС-1 (глушитель) = 3 металлолом + 1 пружина + 1 ткань
Прицел ПУ        = 2 стекло + 2 металлолом + 1 микросхема
Удлинённый магазин = 2 металлолом + 1 пружина
```

**Уникальные моды** (T4–T5) требуют **трофейных деталей**:

```
Коллиматор «Кобра» = 1 коллиматорный модуль (дроп с РБ-босса) + 3 микросхемы + 2 металлолом
Тактический фонарь = 1 LED-блок (дроп с «Прайм-дрона») + 2 батареи
```

**Почему так:**
- Базовые моды — **доступны всем**, рутинный крафт. Игрок чувствует «я могу улучшить любой ствол»
- Уникальные моды — **повод идти на конкретного босса**. Создаёт направленный гринд без перегрузки
- Структурно: рецепт мода — это **обычный recipe** с полем `is_mod: true`, цена в `cost` как у любого крафта

**Альтернатива:** все моды чисто из ресурсов. Минус: пропадает «уникальный мод за уникальный бой».

## Вопрос 3. Снятие мода — заменяемо или фиксировано?

**Контекст:** Игрок поставил ПБС-1 на ТТ. Через 5 уровней нашёл АК-74 — хочет переставить.

### ✅ Рекомендация: **снятие разрешено, шанс поломки мода 20%**

```ts
function unequipMod(weapon: Weapon, modSlot: ModSlot): { mod: Mod | null, destroyed: boolean } {
  const mod = weapon.mods[modSlot];
  if (!mod) return { mod: null, destroyed: false };
  const destroyed = Math.random() < 0.20;
  weapon.mods[modSlot] = null;
  return { mod: destroyed ? null : mod, destroyed };
}
```

UI: при попытке снять — модалка «Снять [ПБС-1] с [ТТ]? Шанс поломки 20%. [Да/Нет]».

**Почему так:**
- Решение «ставить или таскать с собой» — **тактическое**, не одноразовое
- 20% — чувствительная боль, но не парализующая (игрок снимает 5 раз — 1 ломается)
- Активирует **выбор**: «нашёл АК — переставлять ПБС с ТТ или скрафтить новый?»

**Альтернатива A:** снятие без поломки. Минус: моды теряют вес решения, игрок просто двигает их между стволами.
**Альтернатива B:** снятие невозможно (мод привязан к стволу до его поломки). Минус: игрок боится ставить моды на стволы средней руки.

## Вопрос 4. Самоделы при поломке — превращаются во что?

**Контекст:** Самопальная винтовка имеет `durability: 50`. На 0 разваливается. Что игрок видит в инвентаре?

### ✅ Рекомендация: **превращается в 1× `scrap_metal`**

```ts
function onWeaponBreak(weapon: Weapon, inventory: Inventory): void {
  if (weapon.class === "craft") {
    inventory.add("scrap_metal", 1);
    notify(`${itemName(weapon)} развалился. Из обломков добыт металлолом.`);
  } else if (weapon.class === "drop") {
    // дроп-стволы при поломке оставляют свои parts (см. вопрос 5)
    weapon.parts.forEach(part => inventory.add(part.id, 1));
    notify(`${itemName(weapon)} вышел из строя. Детали возвращены.`);
  }
}
```

**Почему так:**
- Самодел = «расходник» по дизайну. Игрок не плачет когда он ломается, потому что **в кармане появляется лом** (немного компенсирует усилия)
- Поддерживает экономику scrap_metal (главный ресурс крафта)
- Дроп-стволы при поломке отдают parts — это даёт **второй путь** добывать запчасти (не только с мобов, но и из «отслуживших» стволов своего же арсенала)

**Альтернатива:** самоделы просто исчезают. Минус: игрок чувствует обиду за потраченные 5 лома.

## Вопрос 5. Лимит на сборку — 1 ствол = 1 экземпляр?

**Контекст:** Игрок собрал 3 АК-74. Это нормально или должен быть лимит?

### ✅ Рекомендация: **множественные экземпляры разрешены, лимит = вес инвентаря**

Структурно:

```ts
// БЫЛО (стек):
{ item_id: "ak_74", count: 3 }

// СТАЛО (отдельные инстансы):
[
  { item_id: "ak_74", instance_id: "ak74_001", mods: { suppressor: "pbs_1" }, durability: 87 },
  { item_id: "ak_74", instance_id: "ak74_002", mods: {}, durability: 100 },
  { item_id: "ak_74", instance_id: "ak74_003", mods: { optic: "kobra" }, durability: 45 }
]
```

Каждый АК-74 — **отдельный инстанс** со своими модами и durability. В UI они отображаются как **3 строки**, а не одна с counter=3.

**Почему так:**
- Это естественно: у двух стволов одного типа РАЗНЫЕ моды и состояния, их нельзя стекать
- Игроку приятно собирать «арсенал» (стелс-АК + штурм-АК + про запас)
- Лимит и так встроен — **вес инвентаря**. АК весит 4 кг, при лимите 50 кг физически невозможно таскать >12 штук

**Импликация:** ресурсы (лом, патроны) — стекаемые. Самоделы T1 расходные (поджига, копьё) — тоже можно стекать (durability у них общая). Дроп-стволы и моды — **обязательно инстансы**.

UI инвентаря должен это поддержать (см. M11.0d).
