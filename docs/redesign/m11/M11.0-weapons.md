## TL;DR

Переходим от 80 «уровневых» предметов к **трёхклассовой системе:**

- 🔨 **Самодел** — крафтится из лома, ломкий, дешёвый, **высокий пиковый урон**, низкая надёжность
- 🎯 **Дроп** — реальное оружие СНГ-эпохи, **собирается из частей** добытых с мобов, надёжное, требует охоты
- 🔧 **Мод** — крафтится отдельно, навешивается на дроп, **меняет характеристики с трейдоффом**

**Принцип:** дроп **никогда не падает целиком**. Только детали (15–25% шанс) → собираешь на верстаке.

Все цифры в этом доке — **черновые**, балансируются в QA pass в конце фазы.

## 1. Item Model (новые поля)

Существующая модель `Item` расширяется. Старые поля сохраняются, новые добавляются через `ItemRegistry` адаптер (см. §11).

```ts
type ItemClass = "craft" | "drop" | "mod" | "part" | "resource" | "ammo" | "consumable" | "broken_craft";
type Caliber = "9x18" | "7.62x25" | "5.45x39" | "7.62x39" | "7.62x54R" | ".308" | "12ga" | "20ga" | "melee" | "thrown";

interface Item {
  // существующие поля
  id: string;
  name_ru: string;
  type: "weapon_melee" | "weapon_ranged" | "armor" | "resource" | "consumable";
  weight: number;
  // ...

  // НОВЫЕ поля (опциональны, ItemRegistry проставляет дефолты)
  itemClass?: ItemClass;            // дефолт: вывести из type
  tier?: 1 | 2 | 3 | 4 | 5;     // дефолт: 1
  caliber?: Caliber;            // только для weapon_ranged и ammo
  capacity?: number;            // ёмкость магазина (только weapon_ranged)
  rate_of_fire?: "single" | "semi" | "burst" | "auto";
  noise?: "silent" | "quiet" | "loud" | "very_loud";
  parts?: string[];             // ID деталей для сборки (только class=drop)
  mod_slots?: number;           // кол-во слотов под модификации (только class=drop)
  durability_max?: number;      // только class=craft (сколько ударов/выстрелов до поломки)
  attached_mods?: string[];     // runtime: какие моды навешены (хранится в save, не в каталоге)
  disassembleRecipe?: RecipeIngredient[];
}
```

### Самодел при поломке → `broken_craft`

**Решение Alex (Q4 в `M11.0-questions-resolved.md`).** Когда `durability_current = 0` у самодела:

1. Оружие **не пропадает**, превращается в `broken_<original_id>` (новый item класса `broken_craft`)
2. Сломанный самодел **не экипируется**, только лежит в инвентаре
3. На верстаке появляется action **«Разобрать сломанное»** (см. §8)
4. Разбор → ~30% от исходного рецепта (округление вниз), генерируется автоматически

**Пример:** «Поджига» (рецепт: 3 scrap + 2 wood + 1 gunpowder, всего 6 единиц).
При поломке → `broken_makeshift_pistol` (вес 0.6 кг, value 0).
Разбор на верстаке → 1 scrap + 1 wood (30% × 6 ≈ 2 единицы, выбираем самые ценные).

**Игровая роль:**
- Recycling loop: «сломал → таскай или выкинь → разбери → крафтани новый»
- Верстак нужен чаще
- Решение «таскать или выкинуть» добавляет вес-как-жадность

## 4. Детали оружия (parts)

Каждая деталь — отдельный предмет в `items.json` с `class: "part"`. Группируется в инвентаре в раздел «Запчасти».

**Пример naming convention:**

```
pm_frame         — Рамка ПМ
pm_slide         — Затвор ПМ
pm_magazine      — Магазин ПМ (8 патронов 9×18)

ak74_barrel      — Ствол АК-74
ak74_receiver    — Ствольная коробка АК-74
ak74_bolt        — Затворная рама АК-74
ak74_magazine    — Магазин АК-74 (30 патронов 5.45×39)
ak74_stock       — Приклад АК-74
```

**Сколько частей на ствол:**

| Тир ствола | Частей |
|---|---|
| T2 пистолеты | 3 |
| T3 ПП/карабины/АКМ | 4 |
| T4 автоматы/снайперки/дробовики | 5 |
| T5 СВД, уникальные | 5 |

**Итого деталей** (грубо): 22 ствола × среднее 4 = **~88 уникальных part-предметов**.

**Часть = одноразовая.** Собрал ствол — части потрачены. Если разобрать (M11.0+) — вернётся часть деталей с шансом.

## 8. Изменения в Workbench (CraftScene)

Сейчас Workbench умеет один режим: «крафт по рецепту». Добавляем два новых режима:

### 8.1 Режим «Собрать оружие» (новый)

UI:
- Список **дроп-стволов**, для которых есть хотя бы 1 деталь
- Для каждого — прогрессбар «3/5 деталей»
- Кнопка «Собрать» активна когда **все детали есть**
- При сборке детали тратятся, в инвентарь падает готовое оружие

### 8.2 Режим «Модификация» (новый, через инвентарь)

UI открывается из инвентаря на конкретном оружии:
- Видны mod_slots (квадратики)
- Можно перетащить совместимый мод из инвентаря в слот
- Tooltip оружия пересчитывается: «ТТ — 7 урона / loud» → «ТТ — 6 урона / quiet [ПБС]»
- Можно снять мод обратно (бесплатно)

### 8.3 Режим «Крафт» (существующий, остаётся)

Без изменений, но рецептов прибавляется:
- Все самоделы (8 шт)
- Все моды (8 шт)
- Базовые расходники

### 8.4 Режим «Разобрать сломанное» (новый)

**Триггер:** игрок выбирает в инвентаре сломанный самодел (`itemClass === "broken_craft"`) и жмёт «Разобрать на верстаке».

**Логика:**
1. `ItemRegistry.computeDisassembleYield(brokenItemId)` возвращает массив `RecipeIngredient[]`
2. Игрок видит превью: «получишь: 1× scrap, 1× wood»
3. Подтверждает → сломанный самодел удаляется, ресурсы зачисляются на склад

**Авто-генерация yield:**
- Берётся `recipe` исходного самодела
- Каждый ингредиент × 0.3, округление вниз
- Если все < 1, гарантируется минимум 1 единица самого ценного ресурса (чтобы разбор всегда что-то давал)

## 9. UI изменения

### 9.1 Инвентарь

- Новая вкладка **«Запчасти»** — отдельно от Оружия/Брони/Ресурсов
- В Запчастях детали **группируются по стволу**: «ПМ (3/3) — собрать» / «АК-74 (2/5)»
- На иконке оружия с навешенными модами — мелкий значок мода в углу
- Tooltip оружия показывает: класс, тир, калибр, урон, точность, шум, mod_slots, навешенные моды

### 9.2 UI лута после боя

- Детали оружия подсвечиваются **оранжевой рамкой** (это редкий лут)
- Моды подсвечиваются **фиолетовой рамкой**
- Над окном лута — счётчик прогресса: «АК-74: было 1/5 → стало 2/5»

### 9.3 Иконки

## 11. ItemRegistry API

Новый файл `src/state/ItemRegistry.ts`. Единая точка чтения предметов.

```ts
// src/state/ItemRegistry.ts
import itemsJson from "../../content/items.json";

class ItemRegistry {
  private items = new Map<string, Item>();

  constructor() {
    for (const raw of itemsJson) {
      this.items.set(raw.id, this.normalize(raw));
    }
  }

  get(id: string): Item | undefined {
    return this.items.get(id);
  }

  // Адаптер: старые предметы получают дефолтные значения новых полей
  private normalize(raw: any): Item {
    return {
      ...raw,
      class: raw.class ?? this.inferClass(raw),
      tier: raw.tier ?? 1,
      mod_slots: raw.mod_slots ?? 0,
      parts: raw.parts ?? [],
      attached_mods: [], // runtime поле, всегда пусто в каталоге
    };
  }

  private inferClass(raw: any): ItemClass {
    if (raw.type === "resource") return "resource";
    if (raw.type === "consumable") return "consumable";
    if (raw.type?.startsWith("weapon")) {
      // старые «выдуманные» оружия → класс craft, надо ребалансить вручную
      return "craft";
    }
    return "resource";
  }
}

export const itemRegistry = new ItemRegistry();
```

**Итог:** старые тесты, которые делают `items.find(i => i.id === "sawed_off")`, продолжают работать. Новые тесты пользуются `itemRegistry.get("sawed_off")` и видят все новые поля.

- `isBrokenCraft(itemId): boolean` — true если itemClass === "broken_craft"
- `computeDisassembleYield(brokenItemId): RecipeIngredient[]` — авто-yield 30% от оригинального рецепта
- `getBrokenVariant(craftItemId): string | null` — для самодела вернуть id сломанной версии (`makeshift_pistol → broken_makeshift_pistol`)

## 12. Save Migration v1 → v2

В `GameState` добавляется функция миграции, вызывается при загрузке сейва.

```ts
// src/state/migrations.ts
export const SAVE_VERSION = 2;

export function migrateSave(save: any): GameState {
  if (!save.version) save.version = 1;

  if (save.version === 1) {
    save = migrateV1toV2(save);
  }

  return save as GameState;
}

function migrateV1toV2(save: any): any {
  // 1. Старые предметы остаются как есть, просто получают дефолтные новые поля через ItemRegistry
  // 2. Удалённые предметы (mutated_gland, prime_circuit) → конвертация:
  for (const stack of save.inventory ?? []) {
    if (stack.item_id === "mutated_gland") {
      stack.item_id = "ak74_bolt";  // легендарный дроп вместо устаревшего ресурса
    }
    if (stack.item_id === "prime_circuit") {
      stack.item_id = "mod_pso1";   // конвертация в редкую модификацию
    }
  }
  // 3. Установить версию
  save.version = 2;
  return save;
}
```

**Тест:** `tests/migration.test.ts` загружает «эталонный сейв v1» из фикстуры и проверяет что после миграции игра запускается, инвентарь не теряется, деньги/уровень/перки сохраняются.
