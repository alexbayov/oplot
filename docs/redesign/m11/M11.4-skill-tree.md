## TL;DR

Заменяем плоские **8 перков** на **дерево из 24 узлов** (3 ветки × 8 узлов), плюс **миграцию** существующих перков.

**Каждый уровень = 1 очко.** 10 уровней → 10 очков → игрок собирает свой билд из 24 узлов. Невозможно прокачать всё.

Узлы открываются **последовательно по ветке** (узел 3 требует узел 2; ветвление внутри ветки минимально для простоты UI).

**50% узлов — пассивные** (работают сразу), **50% — активные** (хуки под M12 «Combat Pass», в M11.4 либо stub-реализация, либо просто пассивная версия).

## 1. Концепция

### 1.1 Три ветки

| Ветка | Идентичность | Что игрок получает | Кого ассоциирует |
|---|---|---|---|
| 🎯 **Стрелок** | Урон, точность, скилы оружия | Бьёт точнее и больнее, чаще критует, владеет огнестрелом | Снайпер из Сталкера, Глеб (Day R) |
| 🛡️ **Выживальщик** | HP, регенерация, сопротивления, расходники | Дольше живёт, лучше использует медикаменты, меньше теряет от ран | Виталий из Day R, классический Survivor |
| 🔧 **Барахольщик** | Крафт, лут, вес, экономика | Больше лута, лучше крафт, может таскать тяжелее, открывает T3-T4 рецепты | Лом-механик, Барон из Сталкера, мастер-кустарь |

### 1.2 Структура ветки

Каждая ветка = **линейная цепочка из 8 узлов**.

```
СТРЕЛОК:
  ┌──┐    ┌──┐    ┌──┐    ┌──┐    ┌──┐    ┌──┐    ┌──┐    ┌──┐
  │1 │───▶│2 │───▶│3 │───▶│4 │───▶│5 │───▶│6 │───▶│7 │───▶│8 │
  └──┘    └──┘    └──┘    └──┘    └──┘    └──┘    └──┘    └──┘
   ▲                       ▲                                ▲
   1 очко                   ramp up                           apex
```

Узел открыт = можно потратить очко.
Узел открывается = все предыдущие в ветке прокачаны.

**Альтернатива (ramified branch)** — рассмотрим в открытых вопросах. На старт делаем линейный.

### 1.3 Очки и уровни

- 10 уровней → **10 очков перка**
- 24 узла → нельзя прокачать всё → **выбор билда**
- На один уровень — выбор: либо новый узел в одной ветке, либо узел в другой (если открыт)

## 3. Active vs Passive узлы

| Ветка | Пассивных | Активных |
|---|---|---|
| Стрелок | 6 | 2 (Прицельный, Двойной выстрел) |
| Выживальщик | 7 | 1 (Второе дыхание) |
| Барахольщик | 8 | 0 |
| **Итого** | **21** | **3** |

### 3.1 Реализация активных узлов в M11.4

Активные узлы **зависят от M12 «Combat Pass»** (где появится UI для активных способностей). На время M11.4:

**Вариант A (рекомендуемый): Stub-implementation**
- Активные узлы **существуют в дереве**, можно прокачать
- Эффект применяется автоматически (без UI кнопки) при триггере:
  - `aimed_shot`: первая атака игрока в бою получает +50% crit и игнорирует укрытие
  - `double_tap`: каждый N-ный ход игрока — 2 атаки автоматом
  - `last_breath`: автоматически срабатывает при HP=0

**Вариант B: вынести в M12**
- В M11.4 — только 21 пассивный узел
- 3 активных узла недоступны до M12

Я бы делал **Вариант A** — игрок видит весь tree, эффект есть, но без полноценного UI. Это **меньший тех долг**, чем «недостроенное дерево».

### 3.2 Заготовки под M12

В коде узла:

```ts
interface SkillNode {
  id: string;
  type: "passive" | "active";
  effect: SkillEffect;
  // Заготовка под M12 — UI hook
  ui_button?: {
    icon: string;
    cooldown_rounds: number;
    trigger: "manual" | "auto";  // в M11.4 все active = "auto"
  };
}
```

→ Когда придёт M12 — просто меняем `trigger: "auto"` → `"manual"` и подключаем UI кнопку.

## 4. Миграция существующих 8 перков

Сейчас в `content/perks.json` 8 перков:

| Старый ID | Старый эффект | Новый узел в дереве |
|---|---|---|
| `tough_skin` | +15 HP | → `hp_1` (Закалка I) |
| `sharp_blade` | +damage melee | → новый узел в дереве (расширим Стрелка ИЛИ отдельный node) — см. вопрос §10.2 |
| `lean_pack` | +5 кг вес | → `weight_1` (Грузчик I) |
| `lucky_scavenger` | +лут | → `extra_loot_1` (Жадные руки I) |
| `keen_eye` | +точность | → `marks_1` (Меткость I) |
| `reinforced_plates` | +броня | → `armor_eff` (Бронежилет) |
| `quick_hands` | -время крафта | → `craft_eff` (Бережливость) — частично |
| `fast_learner` | +XP | → новый узел в Барахольщике или Выживальщике |

**Save migration:**

```ts
function migratePerksToTree(save: GameState): void {
  const oldPerks = save.acquired_perks ?? [];
  save.skill_points_spent = [];

  for (const perkId of oldPerks) {
    const mappedNodeId = PERK_TO_NODE_MAP[perkId];
    if (mappedNodeId) {
      save.skill_points_spent.push(mappedNodeId);
    }
  }

  // Если у игрока было >8 перков (импоссибл, но фейлсейф) — возвращаем очки
  const totalPointsAvailable = save.player_level - 1;
  const pointsUsed = save.skill_points_spent.length;
  save.skill_points_unspent = Math.max(0, totalPointsAvailable - pointsUsed);
}
```

**Принцип:** игрок **не теряет** прогресс. Если у него было 8 перков на 10 уровне — получит 8 узлов и 2 неспента-очка.

## 6. Respec (сброс)

**Игрок должен мочь поменять решение.** Реализуем платный респек.

### 6.1 Правила

- Респек открывается через NPC «Тренер» на базе (когда придёт M14 NPC) ИЛИ сразу как кнопка в SkillTreeScene
- **Полный респек:** возвращает ВСЕ очки, требует:
  - Уровень 5+ (нельзя респечить на старте)
  - Платит **20 единиц основных ресурсов** (лом + электроника + ткань) ИЛИ **редкий расходник «Стимулятор переобучения»**
  - **Cooldown** — 1 респек / 24 часа

### 6.2 UI

```
Полный сброс навыков

Будут возвращены: 7 очков
Цена: 20 лом + 10 электроника + 5 ткань
Cooldown: 24 часа

[СБРОСИТЬ]   [ОТМЕНА]
```

### 6.3 Save migration совместимость

Если игрок респечит — старые перки **не возвращаются** (они уже сконвертированы в узлы дерева). Все потраченные очки возвращаются в `skill_points_unspent`.

## 10. Открытые вопросы

1. **Линейные ветки vs ramified** — у меня в плане каждая ветка = линейная цепочка. Альтернатива: ветка ветвится в узле 4 («Снайпер vs Автоматчик»). Это даёт больше билдов, но **сильно усложняет UI и save migration**. Я бы делал **линейные на старт** + ramified в будущей фазе M13.
2. **Sharp blade (melee damage)** — куда мигрировать? Идеи:
    - Добавить узел `melee_dmg` в Стрелок (но Стрелок про ranged) — плохо тематически
    - Сделать **новую 4-ю ветку «Боец-melee»** — но это удваивает скоуп
    - Растворить в `cover_dmg` (он работает и для melee) — приемлемо
   Я бы сделал **3-й вариант** + дать `cover_dmg` работать и в melee.
3. **Очко на уровень = 1 или 2?** 24 узла на 10 уровней = очень тяжёлый выбор (можно ровно 41% дерева). Альтернатива: 2 очка/уровень = 20 узлов, всё равно нельзя 100% (24). Я бы делал **1 очко/уровень** — тяжёлый выбор = осмысленность билда.
4. **Дополнительные очки за квесты/боссов** — добавлять? Например, убил босса данжа → +1 свободное очко. Это даёт **причину ходить в зачистки** даже когда уже всё надо. Решение зависит от баланса экономики.
5. **Респек cooldown — 24 часа real time vs in-game?** Real time = жёстко для казуалов, in-game time = бесполезно (накачал и переключил). Я бы делал **24 часа real time** + **возможность снять cooldown** редким расходником.
6. **Mid-tree perks для модов** — стоит ли добавить узел типа «Оружейник: +1 mod_slot к любому стволу»? Это сильный узел, может ломать баланс. Лучше не добавлять или оставить как boss-перк (см. вопрос 4).
