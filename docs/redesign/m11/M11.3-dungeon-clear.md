## TL;DR

Зачистка — **endgame-формат сессии** с фокусом, риском и гарантированным жирным лутом в конце.

**Игрок выбирает здание на карте → проходит 5–8 «комнат» подряд → в конце босс с уникальным дропом → выход.**

Отличие от Точки:
- Точка = **бесконечный гринд**, выход когда хочешь
- Зачистка = **линейная последовательность**, отступить = штраф

Зачистка **не повседневная**: cooldown ~6 часов реального времени ИЛИ одноразовый «ключ» расходник.

## 1. Концепция

### 1.1 Что такое «Зачистка»

Зачистка — это **здание** на карте мира с **запертой дверью**. Чтобы войти, нужен **ключ** (расходник) ИЛИ дождаться сброса cooldown-а.

Внутри — **линейный сюжет**: комната → комната → комната → ... → босс → выход.

**Главное отличие от Точки:** игрок **не контролирует** что будет в комнате. Контент **зашит** в дизайн данжа. Это **полу-roguelike**: маршрут предсказуемый, но детали (какие мобы, какое событие, какой сундук) — случайные из пула этого данжа.

### 1.2 Как это играется

1. Игрок жмёт на «Госпиталь» на карте → видит экран превью («Тир: T3 · Босс: Главврач-фанатик · Награда: чертёж медкита T3 · Кулдаун: готово / через X часов»)
2. Жмёт «Войти» → тратит ключ (если есть) ИЛИ запускается cooldown
3. Появляется UI **«Прогресс данжа: ▮▮▯▯▯▯▯▯ (2/8)»** в верхней части экрана
4. Первая комната → бой / событие / сундук → переход к следующей
5. Игрок может **«Отступить»** между комнатами → теряет 50% лута, **boss не считается пройденным**
6. После босса → выходит на базу со всем лутом и **гарантированным боссовым дропом**
7. Cooldown 6 часов реального времени до следующего захода в это же здание

### 1.3 Почему это работает

- **Фокус:** игрок знает «я в данже до конца, нужно дойти до босса»
- **Риск:** отступление = потери, смерть = полная потеря лута
- **Гарантия:** после босса всегда что-то стоящее (не RNG-болото)
- **Планирование:** «иду в Госпиталь за чертежом → нужно подготовиться»

### 2.2 State machine зачистки

```
        ┌─────────────────┐
   ────▶│ DUNGEON_ENTRY   │  ← превью + ключ check
        └──┬──────────────┘
           │ (вход подтверждён)
           ▼
        ┌─────────────────┐
        │   ROOM_INTRO    │  ← painted "вход в комнату N"
        └──┬──────────────┘
           │
           ▼
        ┌─────────────────┐
        │ ROOM_CONTENT    │  ← combat | event | chest
        └──┬──────────────┘
           │ (комната завершена)
           ▼
        ┌─────────────────┐
        │  ROOM_LOOT      │  ← inline loot panel
        └──┬──────────────┘
           │
       ┌───┴────┐
       │        │
  next room?   YES ─▶ ROOM_INTRO (петля)
       │
       NO
       │
       ▼
        ┌─────────────────┐
        │   BOSS_FIGHT    │  ← финальный бой
        └──┬──────────────┘
           │ (босс убит)
           ▼
        ┌─────────────────┐
        │  BOSS_LOOT      │  ← гарантированный + случайный лут
        └──┬──────────────┘
           │
           ▼
        ┌─────────────────┐
        │  EXIT_SUCCESS   │  ← cooldown активируется, на базу
        └─────────────────┘

  Любая комната может закончиться:
    - игрок жмёт "Отступить" → EXIT_RETREAT (-50% лута)
    - игрок умирает → EXIT_DEATH (полная потеря лута)
```

### 2.3 Параметры сессии

```ts
interface DungeonSession {
  dungeonId: string;
  startedAt: number;
  totalRooms: number;       // 5–8 в зависимости от данжа
  currentRoom: number;      // 1..totalRooms+1 (последняя = boss)
  roomsLayout: RoomDef[];   // pre-generated на entry
  lootCollected: ItemStack[];
  state: "entry" | "room_intro" | "room_content" | "room_loot" | "boss_fight" | "boss_loot" | "exit_success" | "exit_retreat" | "exit_death";
}

interface RoomDef {
  type: "combat" | "event" | "chest";
  contentId: string;   // mob_pack_id | event_id | chest_id
}
```

## 3. Room types

### 3.1 Combat room (60% от не-боссовых комнат)

- 1–4 моба, тир соответствует данжу
- Бой в существующем `CombatScene`
- Лут стандартный (mob drop tables)
- **«Сильная комната»** (boss-like): на 5+ ходов, с 3-4 мобами и спец-механикой (например, бесконечный спавн пока не убьёшь «вожака»)

### 3.2 Event room (25%)

Painted scene + **2–4 кнопки выбора**. Каждый выбор имеет **исход**.

**Пример события «Заваленная палата» (Госпиталь):**

```
Painted scene: разрушенная палата, на полу — мёртвый человек в халате, рядом мед-тележка.

Выборы:
[Обыскать врача]      → 80% бинты(2-4), 20% медкит, 10% риск встретить заразного → бой
[Перевернуть тележку] → 100% медикаменты(1-3), 30% шанс на чертёж бинта T2
[Уйти молча]          → нет лута, нет риска, переход к следующей комнате
```

### 3.3 Chest room (15%)

Painted scene + **сундук / тайник** (шкаф, сейф, ящик с инструментами).

- Гарантированный лут тира данжа
- Может требовать **взлома** (мини-игра в будущей фазе M12, пока просто лут)
- Может быть **ловушкой** (5% — взрыв, -20 HP)

## 5. Boss механики (минимум 2-3 на босса)

### 5.1 Базовые механики (общие)

- **Phase change** — боссу при достижении X% HP меняется поведение (summon, mode shift)
- **Reinforcements** — каждые N ходов / при триггере вызывает 1-2 саб-мобов
- **Status application** — наносит статус на хит (bleed / stun / fear) — заготовка под M12

### 5.2 Уникальные механики на босса (M11.3 implements simplified versions)

| Босс | Механика | Реализация в M11.3 |
|---|---|---|
| Главврач-фанатик | Призывает 2 инфицированных при 50% HP | Добавить в `CombatScene` один-разовый trigger по HP-условию |
| Главврач-фанатик | Кровотечение на каждый хит | Прибавить +5 урона на 2 хода (упрощённый bleed без M12 system) |
| Бригадир (фабрика) | Tank mode at 30% HP | Установить флаг `damage_taken_modifier = 0.5` навсегда |
| Бригадир (фабрика) | Вызывает дронов каждые 3 хода | Counter в state, при == 3 → spawn |
| Хозяин тоннелей | Двойной summon (75% и 50% HP) | Два независимых триггера |
| Хозяин тоннелей | Berserk mode | +50% damage, -25% accuracy на оставшийся бой |

**Принцип:** механики в M11.3 — **простые и хардкод**. Полноценная боевая система с активными способностями и статусами — это M12.

## 6. Cooldown / Key система

### 6.1 Cooldown

Каждая зачистка имеет `cooldown_hours: int` в реальном времени (через Yandex SDK `getStorage()`).

```ts
// pseudocode
function canEnterDungeon(dungeonId: string): boolean {
  const lastCleared = getStorage(`dungeon_cleared_${dungeonId}`);
  if (!lastCleared) return true;
  const cooldownMs = dungeons[dungeonId].cooldown_hours * 3600 * 1000;
  return Date.now() - lastCleared >= cooldownMs;
}
```

### 6.2 Entry key (альтернатива cooldown)

Некоторые данжи требуют **расходник-ключ**, без cooldown. Это позволяет:
- Хардкорным игрокам ходить чаще (если фармят ключи)
- Создать **рынок ключей** между NPC (когда придёт M14 narrative)

| Данж | Ключ | Где добывается |
|---|---|---|
| Госпиталь | `hospital_keycard` | Дроп с city_guard 5% |
| Фабрика | `factory_pass` | Дроп с armored_guard 3%, ИЛИ крафт T3 |
| Метро | `metro_token` | Уже существует! Дроп с sniper 8%, crafted T3 |

### 6.3 Combined model (рекомендация)

Использовать **обе**: cooldown + опциональный ключ для **сброса** cooldown-а.

- Без ключа: ждёшь cooldown
- С ключом: ходишь сразу (тратишь ключ)

→ Игроки с большим запасом ключей могут гриндить активнее, без ключей — играют размеренно.

## 11. Открытые вопросы

1. **Динамическая генерация комнат vs фиксированный layout** — генерация на entry даёт реиграбельность; фиксированный layout даёт «выученный данж» (как в Lineage). Я бы делал **гибрид**: типы комнат фиксированы (`combat, event, combat, chest, combat, BOSS`), а **что внутри** комнаты — случайно из пула.
2. **Retreat — где можно?** Только в `room_loot` state? Или и в `room_intro` (до боя)? Я бы разрешал ТОЛЬКО в `room_loot` (после комнаты) — это честнее.
3. **Boss механики — сейчас или сразу с M12?** В M11.3 — хардкодим простые triggers. Когда придёт M12 с боевой системой, **переписываем** на её хуки. Это **тех долг**, но осознанный — за него отвечает M12.
4. **Death — терять ВСЁ или 80%?** Полная потеря = жёстко, может быть отпугивающим. 80% потеря = более «forgiving». Решение зависит от целевого тонуса.
5. **3 данжа достаточно для старта?** Госпиталь, Фабрика, Метро — да. Когда добавлять Бункер, Реактор? Я бы вынес их в **M11.3b «Endgame dungeons»** как опциональную подфазу после основного M11.3.
6. **Painted illustrations для событий** — нужны или можно с фоном данжа + текстовый layer? На старт — текстовый layer достаточно, иллюстрации событий — потом.
