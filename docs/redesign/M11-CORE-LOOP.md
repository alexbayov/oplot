# M11 — «Core Loop Overhaul»

> Версия: v2 (master plan)
> Автор: Zo · PM: Alex
> Дата: 2026-05-28

---

## Цель вехи

Сделать игру **играбельной без сюжета**. Сейчас петля «вылазка → 2–4 боя → возврат» — это **скрипт**, а не игра: игрок не управляет сессией, не выбирает зачем идёт, не понимает что фармит. M11 переписывает ядро: **выбор формата сессии, направленный гринд, осмысленная экипировка, ветка прокачки**.

Сюжет, NPC, дневник, нарратив — **вне скоупа** (откладываем до тех пор, пока не появится самостоятельно желание играть в gameplay-петлю). M11-Narrative из PR #92 — переносим на M14+.

После M11 игрок должен мочь сказать:

> «Мне нужен автомат, чтобы пройти фабрику. Автомат падает с капитанов охраны в городе. Иду фармить капитанов на «Точке: Перекрёсток», по дороге докачаю узел «Стрелок-2». Завтра пойду на «Зачистку: Госпиталь» за чертежом бронника.»

Это и есть «играть хочется» — у действия есть цель, у цели — путь, у пути — выбор.

## Диагноз (что именно сломано)

| Что сейчас | Что должно быть |
|---|---|
| Вылазка = фиксированная цепочка 2–4 боёв, сессия = скрипт | Два формата: **«Точка»** (open-zone фарм с респом) и **«Зачистка»** (линейный данж с боссом) |
| Лут привязан к зоне (`zone.resources`) — фарм всегда «иду в зону и надеюсь» | Лут привязан к мобу (drop table). «Хочу макаров — иду на мародёров». Направленный гринд |
| 80 «выдуманных» предметов без характера | **Три класса экипировки:** 🔨 **Самодел** (крафт из лома, ломкое, дешёвое) + 🎯 **Дроп** (реальные стволы из частей с боссов) + 🔧 **Мод** (крафтятся, навешиваются на дроп) |
| 8 плоских перков (+15 HP, +damage) на 10 уровней | **Дерево навыков** 3 ветки × 8 узлов = 24+ — выбор билда |
| Все предметы «плоско лучше» | Тиры **T1–T5**, привязанные к уровням зон. Видно куда расти |
| `min_damage / max_damage` всё, чем отличается оружие | Калибр + ёмкость + скорострельность + модификации (глушитель, прицел, расширенный магазин) |
| «Дроп оружия целиком 5%» = RNG-фрустрация | **Parts assembly:** боссы дают **детали** (15–25%), игрок собирает ствол на верстаке = прогрессбар вместо лотереи |

## Три класса экипировки (фундамент M11)

| Класс | Откуда | Логика | Тиры | Примеры |
|---|---|---|---|---|
| **🔨 Самодел (craft)** | Только верстак, из лома и базовых ресурсов | Дешёвый, ломкий (X выстрелов/ударов), всегда доступен. Высокий пиковый урон, низкая точность/надёжность. | T1–T2 (часть до T3) | Заточка, поджига, «трубка-винтовка», обрез, копьё, арбалет, граната-самопал |
| **🎯 Дроп (looted/assembled)** | **Только сборка** из частей с конкретных мобов/боссов | Реальное оружие СНГ-эпохи. Стабильное, надёжное. Требует охоты за частями. Целиком с моба **не падает никогда** — только детали. | T2–T5 | ПМ, ТТ, СКС, АКМ, АК-74, СВД, Сайга-12, Ф-1 |
| **🔧 Мод (modded)** | Крафт на верстаке + навешивание на Дроп | Меняет характеристики дропа **с трейдоффом** (не «плоский бонус»). Снимается/переставляется. | T2–T5 | ПБС-1 (глушитель), ПСО-1 (оптика), расширенный магазин, тактическая рукоять |

**Главный принцип:** дроп **никогда не падает целиком**. Капитан охраны не роняет «АК-74 (5%)» — он роняет «затворную раму АК (20%)» или «магазин 5.45 (35%)». Игрок копит 3–5 деталей и собирает ствол на верстаке. **RNG не фрустрирует, потому что прогресс видимый** (3/5 деталей в инвентаре).

**Разборка** (старый ствол → запчасти) — **вне скоупа M11**, добавим в M11.0+ когда у игрока появится явный «избыток дублирующих стволов». Пока: лишний ПМ продаётся торговцу или выкидывается.

## Что НЕ входит в M11 (отдельными PR'ами потом)

- **Combat depth** — таргетинг, сигнатуры мобов, статус-эффекты. → M12 «Combat Pass»
- **Расходники-усилители** (усиленный патрон / стимулятор / дымовая шашка). → M12
- **Разборка оружия** на запчасти + рециклинг старого лута. → M11.0+ или M13
- **Усиление модификаций** (улучшение ПБС → ПБС-2). → M13
- **Энчант** оружия (Lineage-style +1..+N). → M13 (опционально)
- **Нарратив, NPC, дневник.** → M14+ (бывший M11-Narrative из PR #92)
- **Вес-как-жадность** — остаётся в текущем виде, не трогаем. Возможно переосмыслим в M12.

## Правила управления техдолгом (ОБЯЗАТЕЛЬНО)

Чтобы M11 не оставил после себя костыли — три правила, которые **не нарушаем**:

1. **`ItemRegistry` adapter с первого коммита.** Не переписываем `items.json` «в лоб». Вводим `src/state/ItemRegistry.ts`, который читает старый формат и докладывает новые поля (`tier`, `class`, `parts`, `mods`, `caliber`) по дефолту. Старые тесты продолжают работать, новые фичи подключаются постепенно.
2. **Save migration `v1 → v2` в **первом** PR M11.0.** Функция `migrateSave()` в `GameState`. Никаких «потом починим» — миграция пишется одновременно с введением новых полей.
3. **GDD sync обязателен в конце вехи (M11.5).** Не закрываем M11 пока `docs/GDD.md`, `docs/balance.md`, `docs/content-brief.md` не отражают новую систему. Это не косметика — это контракт.

Если эти три правила соблюдены — после M11 проект **чище**, чем до. Если нарушены — техдолг копится, переделка через 2 месяца.

**Обоснование порядка:**

- **M11.0 первым и большим.** Это переразметка контента (`items.json`, `mobs.json`) — фундамент для всего. Внутри строгая последовательность: модель → каталог → части → моды → дропы. Каждый подэтап мержится отдельным PR, ломается одно — остальное живёт.
- **M11.1 второй.** Тиры закрепляют экономику и привязывают мобов к зонам по силе. Без тиров «Точка»/«Зачистка» не знают где какой лут давать.
- **M11.2 раньше M11.3.** Open-zone фарм — это **базовая петля удержания**: зашёл, пофармил, вышел. Игрок будет проводить здесь 70% времени. Зачистки — это «спецоперации» поверх уже работающего фарма.
- **M11.4 предпоследним.** Skill tree имеет смысл когда уже есть направленный гринд (есть зачем качаться) и разные форматы сессий (есть на что тратить перки).
- **M11.5 обязательный финал.** Документация — это контракт. Без неё веха не закрыта.

**Ожидаемая длительность всей вехи:** 5–6 недель focused work.

## Фазы (overview)

### M11.0 — Weapon System Overhaul

**Цель.** Перейти от 80 выдуманных «уровневых» предметов к **трёхклассовой системе** (самодел / дроп / мод) с реальными стволами СНГ-эпохи и сборкой из частей.

**Подфазы:**

#### M11.0a — Items model + ItemRegistry + save migration (3-4 дня)

- Новый `src/state/ItemRegistry.ts` — единая точка чтения предметов с адаптером старого формата
- Новые поля: `class: "craft" | "drop" | "mod" | "part" | "resource" | "ammo"`, `caliber?`, `tier: 1..5`, `parts?: string[]`, `mod_slots?: number`, `durability_max?`
- `migrateSave(v1 → v2)` в `GameState` — старые сейвы остаются играбельны
- Обновление 24 тестов на новую модель (через адаптер)
- **Acceptance:** старые сейвы открываются без потерь, новые поля видны через `ItemRegistry.get(id)`, все тесты зелёные

#### M11.0b — Real weapons catalog + самоделы rebalance (4-5 дней)

- **Каталог дропа (15 стволов):** ПМ, ТТ, Стечкин, ППШ, СКС, АКС-74У, АКМ, АК-74, РПК, СВД, Мосина, ИЖ-43, Сайга-12, Бекас, нож-разведчика НР-43
- **Каталог самоделов (8 предметов, ребаланс):** заточка, поджига, «трубка-винтовка», обрез, копьё, арбалет, граната-самопал «хлопушка», молотов
- **Калибры как отдельные стекаемые ресурсы:** 9×18, 7.62×25, 5.45×39, 7.62×39, .308, 12 калибр, 7.62×54R
- Принцип: самодел — выше пиковый урон, ниже точность/надёжность, дешевле; дроп — стабильнее, надёжнее, требует охоты
- **Acceptance:** игрок видит «ПМ» и «Поджига» как два разных решения для пистолета, а не «плохой/хороший»

#### M11.0c — Parts system + сборка на верстаке (5-6 дней)

- Новый класс предметов: `part` (деталь оружия)
- Каждый дроп-ствол = **3–5 деталей** (пример ПМ: рамка, затвор, магазин = 3 части; АК-74: ствол, ствольная коробка, затворная рама, магазин, приклад = 5 частей)
- Новый режим Workbench: **«Собрать оружие»** — UI с прогрессом «3/5 деталей»
- Рецепты сборки в `content/recipes.json` с новым полем `recipe_type: "assemble"`
- **Acceptance:** игрок видит в инвентаре «затворная рама АК (1/1)», понимает «нужно ещё 4 детали», на верстаке собирает АК-74

#### M11.0d — Mod system (ПБС / оптика / магазин) (3-4 дня)

- Каталог модификаций (минимум 8): ПБС-1, ПБС-универсал, ПСО-1, кратность 4×, расширенный магазин на 9×18, расширенный магазин на 5.45, тактическая рукоять, штык-нож
- Поле `mod_slots: 1..3` у каждого дропа (ПМ — 1 слот, АК-74 — 3 слота)
- UI «навесить мод» в инвентаре на детали оружия
- **Принцип трейдоффа:** ПБС = -1 урон, но «тихий» (в энкаунтерах не привлекает доп-врагов), оптика = +20% точность с дистанции, но -1 ход на прицеливание (заготовка для M12)
- **Acceptance:** игрок может прицепить ПБС на ТТ → видит изменение урона/шума в tooltip

#### M11.0e — Drop tables на мобах + UI лута (3-4 дня)

- Новый формат `mobs.json`:
  ```yaml
  marauder:
    drops:
      - { id: "scrap_metal",    chance: 0.80, count: [1, 3] }
      - { id: "ammo_9x18",      chance: 0.40, count: [3, 8] }
      - { id: "pm_frame",       chance: 0.10 }      # деталь ПМ
      - { id: "leather_jacket", chance: 0.05 }
  captain:
    drops:
      - { id: "ammo_545",       chance: 0.60, count: [10, 20] }
      - { id: "ak74_barrel",    chance: 0.20 }      # ствольная деталь АК-74
      - { id: "pso1_scope",     chance: 0.03 }      # модификация
  ```
- Старая `generateZoneLoot` остаётся как fallback для не-боевых encounter'ов (поиск в развалинах)
- UI окно лута после боя: подсвечивает **детали** оранжевой рамкой (это редкий дроп)
- **Acceptance:** игрок убивает капитана 3 раза, видит «затворная рама АК ×1», «магазин 5.45 ×2», понимает что не хватает ствола и приклада

**Acceptance M11.0 в целом:**
- Все три класса работают (можно скрафтить заточку, собрать ПМ из частей, нацепить ПБС)
- Сейвы мигрируют без потерь
- 248 старых тестов зелёные + ~30 новых на parts/mods/drops
- Документация (`m11/M11.0-weapons.md`) описывает каталог полностью

**Длительность:** 5 PR, ~3 недели.

### M11.1 — Tier System T1–T5

**Цель.** Закрепить иерархию экипировки в **5 тиров**, привязать к уровням зон.

| Tier | Где добывается | Примеры оружия | Примеры брони |
|---|---|---|---|
| **T1** | Лес, подвалы | Заточка, монтировка | Тряпки, ватник |
| **T2** | Пригород, склады | ПМ, СКС, охотничий нож | Кожанка, армейский ремень |
| **T3** | Город, метро | ТТ, АКМ, АКС-74У | Бронежилет «Улей» |
| **T4** | Фабрика, госпиталь | АК-74, Сайга-12, СВД | Армейский броник 6Б |
| **T5** | Финальные зоны (M9 endgame) | РПК, редкие модифицированные | Тяжёлый штурмовой |

- Каждый предмет получает поле `tier: 1..5`
- UI показывает тир (рамка иконки + римская цифра)
- Recipes требуют материалов соответствующего тира (T3 рецепт = нужен T2 материал + редкий компонент)

**Acceptance:**
- Видно куда расти: «у меня T2, пора в город за T3»
- Mob drop tables учитывают тир: ранние мобы не дают T4+ оружие
- Tooltip оружия показывает тир + калибр + модификации

**Длительность:** 1–2 PR, 3 дня.

→ Детали: [`m11/M11.1-tier-system.md`](m11/M11.1-tier-system.md) ✅

### M11.2 — «Точка» (open-zone respawn farming)

**Цель.** Заменить «вылазка = 2-4 боя по скрипту» на **управляемый игроком гринд**.

**Концепт:**

- Игрок выбирает зону → попадает на **«Точку»** (фиксированный спот внутри зоны)
- На точке **волны мобов** появляются с интервалом (по таймеру или по нажатию «Ждать»)
- Каждая волна = 1–3 моба → бой → лут → пауза
- Игрок видит **HUD выхода:** HP / вес / патроны / голод (если будет)
- Кнопка **«Вернуться на базу»** доступна **между волнами**
- Каждые N убийств — шанс **mini-boss spawn** (жирный моб с гарантированным редким лутом)
- На точке есть **визуальный таймер уплотнения**: чем дольше сидишь, тем плотнее и злее волны → риск/ревард

**Зоны с точками** (минимум на старт — 4 точки):

| Точка | Зона | Тир мобов | Что фармить |
|---|---|---|---|
| «Опушка» | Лес | T1 | Лом, шкурки, ранние материалы |
| «Перекрёсток» | Пригород | T2 | ПМ, патроны 9×18, кожанка |
| «Двор-колодец» | Город | T3 | ТТ, АКМ, бронежилет «Улей» |
| «Цех» | Фабрика | T4 | Электроника, АК-74, армейка |

**Acceptance:**
- Игрок САМ решает когда уйти (а не скрипт)
- За одну сессию на точке можно набить лут на 1 апгрейд оружия / брони
- Mini-boss срабатывает не чаще раза в 10 минут активного фарма (телеметрия)
- Возврат с точки сохраняет ВЕСЬ лут (правило веса остаётся как было)

**Длительность:** 3–4 PR, 1.5 недели.

→ Детали: [`m11/M11.2-spot-grinding.md`](m11/M11.2-spot-grinding.md) ✅

### M11.3 — «Зачистка» (линейный данж)

**Цель.** Дать **endgame-формат сессии**: фокус, риск, гарантированный жирный лут в конце.

**Концепт:**

- Игрок выбирает **здание** на карте (госпиталь, фабрика, метро, бункер)
- Внутри — **5–8 «комнат» подряд** (линейная последовательность)
- Каждая комната = **1–3 моба** ИЛИ **событие** (мини-выбор: «открыть сейф / трогать тело / искать аптечку») ИЛИ **сундук** (гарантированный лут T-уровня здания)
- В конце — **гарантированный босс** + **уникальный лут** (чертёж, редкое оружие, модификация)
- Выход **только в конце или через «отступить»** — отступ забирает 50% собранного и весь boss-лут не падает
- **Кулдаун на зачистку:** одно здание = 1 раз в Х часов реального времени (Yandex SDK timer), либо «ключ» как одноразовый расходник

**Зоны с зачистками** (минимум на старт — 3 здания):

| Здание | Тир | Босс | Уникальный лут |
|---|---|---|---|
| «Госпиталь» | T2-T3 | Главврач-фанатик | Чертёж медкита T3 |
| «Фабрика» | T3-T4 | Бригадир смены | Чертёж брони 6Б3 |
| «Метро-станция» | T4-T5 | Хозяин туннелей | Уникальный СВД с модификацией |

**Acceptance:**
- Зачистка ощущается как «событие» (не повседневный фарм)
- Босс — реальный вызов: 2–3 уникальные механики
- Уникальный лут не падает с обычных мобов — только в зачистке
- Кулдаун заставляет планировать «когда пойду»

**Длительность:** 3 PR, 1 неделя.

→ Детали: [`m11/M11.3-dungeon-clear.md`](m11/M11.3-dungeon-clear.md) ✅

### M11.4 — Skill Tree (3 ветки × 8 узлов)

**Цель.** Заменить плоские 8 перков на **дерево выбора билда**.

**Три ветки:**

| Ветка | Идея | Примерные узлы |
|---|---|---|
| **Стрелок** | Урон, точность, скилы оружия | +крит, +точность с дистанции, «прицельный в голову» (актив), «двойной выстрел», -1 ход на перезарядку, +damage в укрытии |
| **Выживальщик** | HP, регенерация, сопротивления | +max HP, регенерация в укрытии, «второе дыхание» при <20% HP, иммунитет к кровотечению, +stamina, -потеря веса при тяжёлой ноше |
| **Барахольщик** | Крафт, лут, экономика | +%лут с мобов, шанс не потратить материал, +вес, разблокировать рецепты T3/T4, +XP за лут, бесплатная попытка модификации в день |

- Старт: каждый уровень = **1 очко перка**
- 10 уровней → 10 очков → игрок выбирает в какие ветки вкладывать
- Узлы открываются по веткам последовательно (нельзя сразу узел 8, надо пройти 1→2→3...)
- **Респек:** платный (ресурсами), 1 раз / N уровней — даёт право на ошибку

**Acceptance:**
- 3 ветки × 8 узлов = 24 перка минимум (vs текущих 8)
- Игрок ощущает разницу между «билдом стрелка» и «билдом барахольщика»
- UI дерева — визуально читаемый, узлы запираются/открываются
- Существующие 8 перков мигрируют в новые узлы (не пропадают)

**Длительность:** 2–3 PR, 1 неделя.

→ Детали: [`m11/M11.4-skill-tree.md`](m11/M11.4-skill-tree.md) ✅

## Риски и митигация

| Риск | Митигация |
|---|---|
| Реальные стволы → юридические/политические претензии Яндекса | Это **постсоветская атмосфера**, стволы — часть сеттинга (Сталкер, Метро 2033, Day R уже на YI). Если будет flag — переименуем («пистолет 9мм» вместо «ПМ», но характеристики сохраним). |
| Parts система → инвентарь захламляется деталями | UI группировки: детали одного ствола склеиваются в «комплект ПМ (3/5)» в отдельной вкладке инвентаря «Запчасти». |
| Точка с респом = бесконечный гринд → дисбаланс | Лимиты: вес, патроны, HP, mini-boss как «потолок». Логи телеметрии в M11.2 покажут что душить. |
| Зачистки могут «съесть» Точку (игрок ходит только в данжи) | Cooldown на зачистки + уникальный лут только там = зачистки редкие. Точка = ежедневный хлеб. |
| Skill tree без combat depth (M12) скучный | 50% узлов в M11.4 — пассивные (работают сразу). Активные узлы заложим как ХУКИ, доделаем в M12 когда придут активные способности оружия. |
| Рост размера билда (новые иконки оружия + деталей + модов) | Иконки — 64×64 painted, ~3 KB каждая. ~60 новых иконок ≈ 180 KB. Не критично для лимита Яндекса. |
| Save migration ломает старые сейвы | Миграция в первом PR M11.0a + автотест на загрузку «эталонного старого сейва». |

## Acceptance вехи в целом

После M11 игрок может:

1. Выбрать формат сессии: **«сегодня пофармлю на Точке»** или **«иду на зачистку Госпиталя»**
2. Назвать конкретное оружие и сказать **зачем оно ему** («хочу АК-74 — он бьёт 5.45×39, пробивает броню»)
3. Назвать конкретного моба и сказать **что с него падает** («с капитана охраны падает затворная рама АК-74 с шансом 20%»)
4. Описать **свой билд:** «я качаю Стрелка → +крит → потом возьму прицельный в голову»
5. Объяснить **зачем ходит каждый день:** «на Точке нужны материалы для бронника T3»
6. Объяснить **прогресс сборки:** «собрал ПМ полностью, на АК-74 не хватает приклада и магазина»

Если хотя бы 4 из 6 — веха закрыта.
