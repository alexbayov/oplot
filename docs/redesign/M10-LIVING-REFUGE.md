## Цель вехи

Превратить Оплот из «приложения с четырьмя вкладками» в **живое убежище внутри живого мира**.

Это **не контент-патч**. Это смена парадигмы UX: ассеты у нас уже AAA-уровня (painted hero, painted мобы, 9 painted зон, painted UI), но **архитектура взаимодействия** до сих пор «menu → submenu → submenu». Пока игрок видит вкладки, а не убежище — он не чувствует что во что-то инвестировал. Никакой апгрейд боёвки/шмота это не починит.

## Диагноз (что именно сломано)

| Что сейчас | Что должно быть |
|---|---|
| 4 вкладки справа от портрета | Убежище — одна сцена в разрезе с интерактивными объектами |
| Карта зон = 3×3 grid карточек | Painted карта пустоши с pin-локациями и тропами |
| Вылазка = клик «Глубина 1» → сразу бой | Путь до боя — 2–3 текстовых encounter'а с выбором |
| Возврат на базу = телепорт + банер «Победа» | Сцена возвращения: герой заходит, бросает рюкзак, садится у огня |
| Радио в отдельной вкладке, к которой надо тянуться | Радио на стене базы. Шипит и мигает само. Часть пейзажа |
| Мастерская в отдельной вкладке без визуала | Верстак в углу базы. Видны заготовки. Видно прогресс |

Все 4 пункта работают вместе: **когда они есть — игрок чувствует место и путь. Когда их нет — он листает меню.**

## Порядок фаз (обновлено после M10.0)

После ревью M10 v1 добавлена фаза **M10.0 Foundation** и **рекомендуемый порядок изменён**.

```
M10.0  Foundation        (1 PR,  1 день)   ← телеметрия, sceneStack, baseline screenshots
   ↓
M10.3  Base as Scene     (3-4 PR, 1 неделя) ← высший leverage (база видна каждый запуск)
   ↓
M10.1  World Map         (2 PR,  3 дня)
   ↓
M10.4  Return Ritual     (1-2 PR, 2 дня)   ← переиспользует painted interior из M10.3
   ↓
M10.2  Encounters        (3 PR,  1 неделя) ← gameplay-расширение в готовый визуальный мир
```

**Обоснование смены порядка:**

- **M10.0 первым** — без baseline-телеметрии нельзя измерить успех остальных фаз. Без решения по scene stack — M10.3 встанет на полпути. См. [`m10/M10.0-foundation.md`](m10/M10.0-foundation.md).
- **M10.3 раньше M10.1** — базу игрок видит **N×10 чаще, чем карту** (каждый запуск, каждый возврат). Максимальный эмоциональный сдвиг на сессию. Карта видна 1× на цикл вылазки.
- **M10.2 (encounters) последним** — это gameplay-расширение, оно встанет на готовый «живой мир». Если делать раньше — encounters будут жить в визуально-старом мире, потом всё равно перерисовывать.
- **M10.4 после M10.3** — Return Ritual переиспользует painted interior из M10.3. Делать раньше = двойная работа.

**Связанные документы:**
- [`m10/M10.0-foundation.md`](m10/M10.0-foundation.md) — спека foundation-фазы (телеметрия, sceneStack, baseline)

### M10.2 — Journey Encounters (события в вылазке)

**Цель.** Между картой и первым боем — 2–3 коротких textual events с выбором. Превращает вылазку из «зашёл-вышел» в roguelike-журну.

**Формат encounter'а.**
```
text: "У старого пенька лежит обугленный рюкзак.
       Внутри что-то блестит, но воняет тленом."
illustration: optional small painted image
choices:
  - text: "Обыскать"
    outcome: +1 scrap, +1 cloth, -3 HP (риск отравления)
  - text: "Обойти"
    outcome: ничего, +1 ход к следующему событию
  - text: "Сжечь"
    outcome: -1 wood, +0 loot, но удаляет mob_ambush flag
```

**Контент.** 30–50 уникальных encounter'ов разбитых по зонам. Лес = свои (лесные ловушки, звери, заброшенный лагерь), Метро = свои (затопленный туннель, граффити, забаррикадированный вагон), и т.д.

**Ассеты:**
- 30–50 коротких текстов в `content/encounters.json`
- Опционально: 10–15 painted small illustrations 256×256 для самых атмосферных событий

**Код:**
- Новый `src/systems/encounters.ts` — pool по зоне, выбор RNG, разрешение outcome
- Новая `src/scenes/EncounterScene.ts` — рендер текста + выборов + outcome
- Изменить flow: `SortieScene` → 2-3× `EncounterScene` → `CombatScene` → `LootScene` → `ReturnScene`

**Длительность:** 3 PR (1 контент структура, 1 движок + сцена, 1 первые 15-20 encounter'ов).

**Acceptance:**
- [ ] Каждая вылазка содержит 1-3 encounter'а перед боем
- [ ] Минимум 20 уникальных encounter'ов на момент мержа
- [ ] Outcome влияет на статус (HP, ресурсы, флаги) и видно игроку
- [ ] Encounter'ы фильтруются по зоне
- [ ] Тесты на encounter resolution: pure, RNG seeded, без побочек

### M10.3 — Base as Scene (убежище как сцена)

**Цель.** `BaseScene` перестаёт быть «портрет слева + меню справа». Становится painted side-view убежища с интерактивными объектами.

**Что в кадре.** Одна painted illustration убежища в разрезе. На ней:
- **Стол с бумажной картой** → клик = открыть World Map
- **Верстак с инструментами** → клик = Мастерская (overlay над сценой)
- **Радио на полке** → клик = Радио. Когда есть новый сигнал — мигает
- **Раскладушка / лежанка героя** → клик = карточка героя (HP, XP, перки)
- **Сундук в углу** → клик = инвентарь
- **Дверь** → клик = «Выйти на вылазку» (если карта уже выбрана) или подсветка стола с картой
- **Окно** → не интерактивно. За окном painted view пустоши, меняется по дню/ночи (на будущее)
- **Лампа / костёр** в центре — атмосферный source света + dust particles в свете

Все интерфейсы открываются **поверх** базы (полупрозрачно), чтобы пейзаж убежища оставался виден. **Никаких полностью отдельных сцен с тёмным фоном.**

**Ассеты:**
- `assets/backgrounds/base_interior.png` — 1280×720 painted refuge interior
- Опционально: separate hotspot overlay images для подсветки на hover

**Код:**
- Переписать `src/scenes/BaseScene.ts` — hotspot-based. Без правой колонки меню
- Переписать `InventoryScene`, `CraftScene`, `RadioScene`, `ProgressionScene` чтобы открывались как overlay над BaseScene (не replacing scene)
- Альтернатива (проще): они продолжают быть отдельными сценами, но при возврате — fade-back на ту же BaseScene без перерисовки

**Длительность:** 4 PR (1 ассет, 1 hotspot foundation, 2 — overlay-режим для остальных сцен).

**Acceptance:**
- [ ] BaseScene = painted illustration без правой колонки меню
- [ ] 5+ интерактивных hotspots
- [ ] Radio mig flag (визуальный пинг) когда есть unread сигнал
- [ ] Все subscene открываются и закрываются плавно
- [ ] Vignette + dust + ambient particles работают (это уже есть в M9)

### M10.4 — Return Ritual (ритуал возвращения)

**Цель.** После победы — короткая сцена возвращения. 3-5 секунд эмоционального ритма.

**Что в кадре.**
1. Fade-in painted base interior (как в M10.3)
2. Герой-силуэт входит через дверь (tween)
3. Рюкзак падает на пол — лут визуально валится (icons этих item'ов появляются и оседают)
4. Герой садится у лампы
5. Если есть unread radio signal — радио начинает шипеть (sound future)
6. Появляется кнопка «Разобрать рюкзак» → переход в LootScene (взять/выкинуть/положить в сундук)

**Ассеты:**
- 1-2 painted silhouettes героя (нормальный, раненый — выбирается по HP%)
- Существующие icon ассеты предметов (уже есть)

**Код:**
- Расширить `ReturnScene.ts` — добавить mini-cinematic с tween'ами
- Лут позиционируется в animation на полу
- Логика: после Combat victory → ReturnScene → cinematic → LootScene

**Длительность:** 1-2 PR.

**Acceptance:**
- [ ] После победы есть 3-5 сек сцена возвращения
- [ ] Силуэт героя анимируется в кадре
- [ ] Лут визуально оседает на полу
- [ ] Можно skip кликом
- [ ] Раненый герой (HP<30%) хромает / медленнее
- [ ] Раздалившееся радио (рандом 30%) триггерится после возвращения

## Метрика успеха

**Subjective (главное):** после прохождения M10 хочется запустить ещё одну вылазку. Сейчас — не хочется. Это решающий критерий.

**Objective:**
- Средняя длительность сессии (телеметрия через Yandex SDK) растёт
- Day-2 retention растёт
- В playtest'ах с друзьями/тестерами — комментарии вида «куда нажать что бы…» сменяются на «о, тут можно…»

## Зависимости / риски

| Риск | Митигация |
|---|---|
| Painted illustration базы выходит «не той» — переделывать 3 раза | Перед production-ассетом сначала 3-4 варианта через Gemini, выбираем |
| Encounters получаются monotone | 30+ уникальных, ревью каждого pool'а на разнообразие outcome'ов |
| Hotspot-режим базы конфликтует с существующими сценами | Сначала **попробовать overlay-mode как чистая надстройка** без переписывания subscene |
| Сцена возвращения раздражает на N-ной вылазке | Skip-by-click + skip-by-default после первых 3 успешных вылазок |
