# «Оплот» — большой план редизайна UI/UX и превращения прототипа в цельную игру

## 0. Диагноз после просмотра проекта

Я посмотрел репозиторий `alexbayov/oplot`, README, GDD, style-guide, текущий `docs/ui-ux-plan.md`, сцены Phaser и живую игру в браузере. Стек: Phaser 3 + TypeScript + Vite, mobile-first 360×640, Яндекс.Игры, survival RPG loop «Оплот → вылазка → бой → лут → возврат → крафт».

Главный вывод: игра уже имеет рабочий скелет и контент, но визуально воспринимается как debug-прототип. Проблема не в одной кнопке или одной иконке, а в том, что отсутствует цельная presentation layer:

- нет сильной первой сцены и ощущения «я в убежище»;
- UI не рассказывает историю мира;
- боевой экран не даёт тактической читаемости;
- карта не ощущается как карта, а как список карточек;
- инвентарь/крафт не ощущаются как физические предметы и добыча;
- прогрессия и радио не имеют драматургии;
- ассеты есть, но используются как небольшие вставки, а не как композиционная основа;
- кнопки выглядят как системные прямоугольники, не как элементы постапокалиптического survival-интерфейса;
- текущий `docs/ui-ux-plan.md` полезен как локальный UI-pass, но слишком мал для превращения игры в полноценный продукт.

Цель редизайна: не «приукрасить», а пересобрать восприятие игры так, чтобы игрок за первые 30 секунд понял: это мрачная survival RPG про риск, вес, добычу, бой и возвращение домой.

## 1. Целевая фантазия продукта

### Что игрок должен чувствовать

1. **Оплот — это единственное безопасное место.**
   Главный экран должен быть не меню, а убежищем: темнота, стол, радио, рюкзак, оружие, склад, потрёпанная карта.

2. **Вылазка — это риск.**
   Перед выходом игрок должен чувствовать ставку: глубина, вес, враги, газ, шанс награды, шанс смерти.

3. **Бой — это читаемое напряжение.**
   Сейчас бой выглядит как панель статов. Нужно поле столкновения: герой слева, враги справа, очередь, состояние, действие, последствия.

4. **Лут — это жадность.**
   Экран добычи должен быть одним из самых вкусных экранов: предметы, вес, редкость, «взять ещё или уходить».

5. **Крафт — это мастерская выжившего.**
   Не список рецептов, а верстак: результат, компоненты, дефицит, улучшение, ощущение сборки.

6. **Радио — это неопределённость.**
   Не список сообщений, а шум эфира, подозрительные сигналы, выбор доверять или нет.

7. **Прогрессия — это взросление персонажа.**
   Уровни/перки должны выглядеть как досье выжившего, а не как список бонусов.

## 2. Новая арт-дирекция

Текущий style-guide уже задаёт «Military Graphic Novel». Его надо усилить и превратить в production bible.

### 2.1. Визуальный язык

Основной стиль:

- мрачный modern post-apocalypse;
- толстые контуры на персонажах/предметах;
- грязная оливково-песочная палитра;
- минимум чистых ярких цветов;
- световые акценты только на действиях, опасности, редкости, важных решениях;
- интерфейс будто собран из военных планшетов, изоленты, бумажных карточек, ржавых бирок, старого радио.

Нельзя:

- пиксель-арт «под денди», если цель не ретро;
- web2.0-градиенты;
- мультяшные круглые кнопки;
- emoji как игровые иконки;
- «чистые» UI-панели без материала;
- случайные шрифты и размеры.

### 2.2. Материалы интерфейса

Сделать UI-kit из 5 материалов:

1. **Металл/бронепластина** — основные панели, боевые карточки, HP/броня.
2. **Пыльная бумага/карта** — карта зон, заметки, радио-сводки.
3. **Изолента/маркер** — активные CTA, выбранные вкладки, предупреждения.
4. **Рюкзачная ткань** — инвентарь, слоты, экипировка.
5. **Экран старого прибора** — радио, лог боя, системные сообщения.

### 2.3. Цветовые токены

Оставить текущую базу, но расширить:

- background: `#111210`;
- panel: `#252620`;
- panel-light: `#323329`;
- border: `#55513E`;
- text-main: `#D4C5A0`;
- text-muted: `#8A8070`;
- danger: `#8B0000`;
- warning: `#FFB300`;
- action: `#C5A267`;
- stamina/weight: `#4682B4`;
- poison/gas: `#6F8A4D`;
- locked: `#5A5A55`;

В коде это должно стать `src/ui/tokens.ts`, а не разрозненные hex в сценах.

### 2.4. Шрифты

Нужны 3 уровня:

- заголовки: Oswald / Bebas Neue / Russo One;
- основной текст: PT Sans / Roboto Condensed;
- цифры/лог/радио: Roboto Mono / Share Tech Mono.

Важно: на 360×640 текст должен быть крупнее, чем сейчас. Мелкий `9px–11px` оставить только для второстепенных подписей. Основные решения — 14–18px.

## 3. Архитектурная цель редизайна

Сейчас сцены напрямую рисуют Phaser objects. Для полноценной игры нужно ввести UI-layer:

```text
src/ui/
  tokens.ts
  layout.ts
  components/
    Panel.ts
    Button.ts
    IconButton.ts
    StatBar.ts
    ItemSlot.ts
    ItemGrid.ts
    Tooltip.ts
    Toast.ts
    Modal.ts
    BottomNav.ts
    SceneHeader.ts
    RiskMeter.ts
    RewardCard.ts
    CombatantCard.ts
    RadioSignalCard.ts
```

Задача: сцены должны описывать смысл, а UI-компоненты — отвечать за стиль, размеры, hover/touch, disabled, selected, danger, accent.

Пример целевого подхода:

```ts
createSceneHeader(this, {
  title: "Карта",
  subtitle: "Выбери маршрут",
  backTo: "BaseScene",
});

createZoneCard(this, {
  zone,
  unlocked,
  risk,
  rewardPreview,
  onEnter,
});
```

А не руками расставлять `text`, `rectangle`, `image` в каждой сцене.

## 4. Редизайн экранов

### 4.1. Boot / Loading

Сейчас загрузка функциональная. Нужно сделать первый контакт с миром.

Что сделать:

- фон: затемнённые руины/убежище;
- логотип «ОПЛОТ» с коротким слоганом: «Выжить. Унести. Вернуться.»;
- прогресс загрузки ассетов;
- короткие survival tips:
  - «Лишний килограмм может стоить хода.»
  - «Не каждый сигнал по радио — помощь.»
  - «Лучший лут бесполезен, если ты не вернулся.»
- обработка ошибок загрузки понятным экраном: «Не удалось загрузить контент», кнопка «Повторить».

Критерий качества: игрок ещё до меню понимает тон игры.

### 4.2. BaseScene — Оплот

Сейчас: карточка героя + кнопки. Работает, но не выглядит как база.

Цель: главный экран должен быть «комнатой выжившего».

Новая композиция:

```text
┌────────────────────────────┐
│ ОПЛОТ          день 12 / ночь│
│ [портрет героя] HP / XP / вес│
│                              │
│     затемнённое убежище       │
│  стол, карта, радио, рюкзак   │
│                              │
│ [В ВЫЛАЗКУ] — главный CTA     │
│                              │
│ [Склад] [Мастерская]          │
│ [Радио] [Герой]               │
│                              │
│ нижняя навигация              │
└────────────────────────────┘
```

Что добавить:

- фоновый арт «убежище» вместо `bg_forest`;
- герой не маленькой иконкой, а портретом/силуэтом;
- карточка состояния:
  - HP;
  - уровень;
  - XP до уровня;
  - текущий вес/лимит;
  - оружие/броня;
  - статус радиосигналов;
  - доступные дейли/награды;
- главный CTA «В вылазку» должен быть самым заметным объектом;
- кнопки вторичных систем не одинаковые: иконка + подпись + маленький индикатор:
  - Инвентарь: заполненность склада;
  - Мастерская: доступно рецептов;
  - Радио: новые сигналы;
  - Прогрессия: перки/XP.

UX:

- первый запуск: подсветить «В вылазку»;
- после возвращения: показать summary «Принесено: X кг / Y предметов»;
- если есть новый уровень: кнопка «Герой» пульсирует;
- если есть новые радио-сигналы: радио шумит/мигает.

### 4.3. MapScene — карта мира

Сейчас: вертикальный список зон с карточками. Это понятно, но не карта.

Цель: игрок выбирает маршрут, а не строку в списке.

Новая композиция:

- фон: физическая карта города/окрестностей;
- зоны как точки/карточки на карте;
- закрытые зоны затемнены и соединены пунктирными маршрутами;
- каждая зона имеет:
  - название;
  - уровень;
  - риск;
  - тип угрозы: люди / мутанты / газ / дроны;
  - уникальные ресурсы;
  - статус босса/дейли;
  - прогресс глубин.

Карточка зоны:

```text
ЛЕС
ур. 1 · риск низкий
Ресурсы: дерево, ткань, еда
Враги: пёс, мародёр
[Войти] [Босс/Дейли]
```

Для закрытых:

```text
СКЛАД
Закрыто
Нужно: завершить Лес depth 2
[Как открыть]
```

UX:

- нажал зону → открывается detail panel, не сразу старт;
- detail panel показывает награды и опасность;
- только потом «Начать вылазку»;
- для mobile удобно: один большой вертикальный route-map можно скроллить.

### 4.4. SortieScene — подготовка к вылазке

Сейчас: три кнопки depth. Это слишком сухо.

Цель: сделать экран ставки.

Новая структура:

```text
ВЫЛАЗКА: ЛЕС
фон зоны

Глубина 1  [низкий риск]  2 боя  ресурсы T1
Глубина 2  [средний риск] 3 боя  больше лута
Глубина 3  [высокий риск] 4 боя  босс/редкий шанс

Снаряжение:
Нож · Куртка · Аптечки x2 · Патроны x4

Предупреждения:
Вес влияет на инициативу

[НАЧАТЬ]
```

Добавить:

- risk meter для каждой глубины;
- preview врагов;
- preview ресурсов;
- требуемый уровень;
- предупреждение про газ;
- подсказку «чем глубже — тем больше боёв и возврат дольше».

Ключ: выбор depth должен быть понятным и эмоциональным.

### 4.5. CombatScene — бой

Это главный проблемный экран. Сейчас он выглядит как таблица HP + лог + кнопки. Нужно превратить в сцену боя.

Цель: за 1 секунду игрок должен понимать:

- кто ходит;
- кто враг;
- сколько HP;
- что сделал враг;
- какие действия доступны;
- почему действие недоступно;
- где риск смерти.

Новая композиция:

```text
┌────────────────────────────┐
│ Очередь: Герой > Пёс > ...  │
│                              │
│        [враги справа]        │
│  фон зоны: лес/склад/город   │
│ [герой слева]                │
│                              │
│ HP герой / вес / броня       │
│ HP врагов + статусы          │
│                              │
│ Лог последнего события       │
│                              │
│ [АТАКА] [УКРЫТИЕ] [АПТЕЧКА]  │
│ [ОТСТУПИТЬ]                  │
└────────────────────────────┘
```

Что сделать:

1. **Боевой stage**
   - фон текущей зоны;
   - герой слева;
   - враги справа группой;
   - мёртвые враги затемняются/падают;
   - у выбранной цели подсветка.

2. **Targeting**
   Сейчас «Атака» не даёт явного выбора цели. Нужно:
   - нажал «Атака» → подсвечиваются враги;
   - тап по врагу → удар;
   - если враг один — автоцель, но всё равно показать подсветку.

3. **Turn order**
   Очередь ходов сверху: маленькие портреты/иконки.
   Вес героя визуально двигает его назад в очереди.

4. **Лог боя**
   Лог должен показывать только 2–3 последних события, крупно:
   - «Пёс кусает: -8 HP»
   - «Ты уходишь в укрытие»
   - «Мутант промахнулся»

5. **Состояния**
   Над врагами:
   - HP bar;
   - armor/defense icon;
   - cover icon;
   - phase 2 boss icon;
   - gas damage indicator.

6. **Анимации**
   - удар героя: рывок/flash;
   - урон: красный floating text;
   - промах: серый `MISS`;
   - лечение: зелёный pulse;
   - смерть: fade + collapse;
   - phase transition boss: экранная тряска + красный flash.

7. **Кнопки**
   Кнопки должны быть большими, не «дендиковскими прямоугольниками»:
   - иконка + текст;
   - disabled reason;
   - press feedback;
   - cooldown/ресурс под кнопкой:
     - Аптечка x2;
     - Патроны x4;
     - Укрытие активировано.

Критерий качества: бой можно понять по скриншоту без чтения кода.

### 4.6. LootScene — добыча

Сейчас: сетка уже лучше, но экран не продаёт главную механику «жадность vs выживание».

Цель: лут должен быть моментом кайфа и тревоги.

Новая структура:

```text
ДОБЫЧА

[предметы как физические карточки/слоты]

Вес: 12.5 / 30 кг
Скорость возврата: +18%
Инициатива в следующем бою: -7

[Взять всё]
[Выбрать вручную]
[Оставить]
[Вернуться]
```

Добавить:

- предметы с rarity border;
- показывать вес каждого предмета;
- tap по предмету → взять/убрать;
- long press/tooltip → описание/flavor;
- авто-сортировка:
  - ресурсы;
  - медикаменты;
  - оружие;
  - редкое;
- индикатор перегруза:
  - синий → норм;
  - жёлтый → тяжело;
  - красный → перегруз;
- прямо писать последствия:
  - «Ты станешь медленнее на 12 инициативы»;
  - «Возврат займёт 34 сек».

Главный UX: игрок должен осознанно мучиться — взять металл или аптечку.

### 4.7. InventoryScene — инвентарь/склад

Сейчас: базовая grid-сетка есть. Нужно сделать полноценный survival inventory.

Цель: инвентарь должен быть «рюкзаком», а не таблицей.

Разделы:

1. **Экипировка**
   - оружие;
   - броня;
   - маска/утилити;
   - расходники для вылазки.

2. **Рюкзак/склад**
   - grid slots;
   - фильтры: всё / ресурсы / оружие / броня / медицина / патроны;
   - сортировка: вес / тир / зона / количество;
   - поиск не нужен для mobile, лучше фильтры.

3. **Item detail panel**
   При тапе:
   - большая иконка;
   - название;
   - тир;
   - вес;
   - описание;
   - где добывается;
   - используется в рецептах;
   - кнопки: экипировать / разобрать / выбросить / закрыть.

4. **Вес**
   - общий вес;
   - вес по категориям;
   - предупреждение «тяжёлый склад» не важно, но «рюкзак вылазки» важно.

В будущем можно добавить «Tarkov-lite grid», где предметы занимают 1×1, 1×2, 2×2, но это отдельная большая механика. Для текущей версии достаточно визуальной сетки.

### 4.8. CraftScene — мастерская

Сейчас: список рецептов с ингредиентами текстом. Нужно сделать верстак.

Цель: крафт должен ощущаться как сборка оружия/брони из найденного мусора.

Новая структура:

```text
МАСТЕРСКАЯ

Категории: Оружие / Броня / Медицина / Боеприпасы / Утилити

[карточка рецепта]
иконка результата
название + тир
статы до/после
компоненты и хватает/не хватает
[СОЗДАТЬ]
```

Добавить:

- визуальные компоненты вместо строки текста;
- missing ingredients красным;
- «можно создать» карточки наверх;
- preview изменения статов:
  - урон +5;
  - вес +1.2;
  - защита +2;
- анимация сборки:
  - детали сходятся в центр;
  - вспышка;
  - итоговый предмет.

Будущая большая фича: модульная сборка оружия.

### 4.9. ReturnScene — возвращение

Сейчас: прогресс-бар возврата. Нужна драматургия.

Цель: возвращение должно быть напряжением после лута.

Сделать:

- герой/силуэт идёт по маршруту;
- фон зоны затемняется;
- weight penalty виден;
- события по дороге:
  - «Шорох в кустах…»
  - «Рюкзак режет плечо»
  - «Слышен далёкий выстрел»
- прогресс «до Оплота»;
- итоговый экран:
  - добыто;
  - XP;
  - новый уровень;
  - открыты зоны;
  - радио сигнал обновлён;
  - кнопка «Разобрать добычу».

Монетизация `×2 лут` должна быть встроена аккуратно:

- показывать после результата;
- не до завершения эмоции;
- текст: «Усилить добычу ×2»;
- не ломать атмосферу.

### 4.10. RadioScene — радио

Сейчас: текстовые карточки. Нужно сделать мистику и риск.

Цель: радио — отдельная мини-драма доверия.

Новая структура:

```text
РАДИО
шум, волны, старая шкала частоты

Доверие: -2 / +5

[сигнал]
источник: неизвестный
зона: склад
тип: просьба / ловушка? / слух?
истекает через 2 вылазки

[Ответить] [Игнорировать]
```

Добавить:

- фон `radio_panel_bg`;
- waveform animation;
- иконки truth/trap/ambiguous, но НЕ раскрывать правду напрямую;
- доверие как шкала;
- outcome как отдельный экран:
  - награда;
  - засада;
  - доверие изменилось;
  - последствия.

Важно: не писать игроку «trap» явно, если это должно быть неопределённостью. Иконки должны быть внутренними подсказками, не спойлерами.

### 4.11. Progression / LevelUp

Сейчас: функционально, но сухо.

Цель: герой должен ощущаться как растущий выживший.

ProgressionScene:

- экран «Досье»;
- портрет героя;
- уровень;
- XP bar;
- список перков как жетоны/нашивки;
- суммарные бонусы;
- история последних достижений.

LevelUpScene:

- full-screen moment;
- «Новый уровень» с сильной анимацией;
- 3 карточки перков;
- каждая карточка:
  - иконка;
  - название;
  - короткий эффект;
  - flavor;
  - тип: бой / лут / вес / крафт.

После выбора:

- карточка вспыхивает;
- добавляется в досье;
- игрок возвращается туда, где был.

### 4.12. Monetization UX

В игре уже есть rewarded/interstitial/IAP. Визуально их нужно подружить с миром.

Правила:

- реклама не должна выглядеть как внезапный web popup;
- rewarded должен быть «радио-помощь», «торговец», «сброс маршрута», «удвоить добычу»;
- interstitial только после завершённого цикла, не перед важным действием;
- IAP как «припасы от каравана», а не магазин с конфетами.

Экран магазина:

- Starter Pack = «Набор выжившего»;
- Gas Pack = «Фильтры и антидот»;
- Ads Remover = «Премиум-снабжение / без перебоев связи»;
- цены через Yandex Payments;
- восстановление покупок;
- понятная кнопка закрытия.

## 6. Mobile UX

Платформа mobile-first, значит дизайн должен строиться от пальца.

Правила:

- основная зона тапа минимум 48×48;
- важные действия в нижней трети экрана;
- не использовать hover как обязательный UX;
- tooltip на mobile = tap detail, не pointerover;
- все сцены должны помещаться в 360×640 или иметь понятный scroll;
- нижний safe-area под баннеры Яндекс.Игр;
- критические кнопки не ставить под sticky banner;
- тексты не меньше 12px, ключевые — 14–18px;
- длинные описания открывать в modal, не пытаться втиснуть.

## 10. Главные принципы, чтобы не скатиться обратно

1. **Каждый экран имеет одну главную мысль.**
   Base = безопасность. Map = выбор риска. Combat = выживание. Loot = жадность. Craft = улучшение.

2. **Не рисовать UI вручную в каждой сцене.**
   Всё через компоненты.

3. **Каждая кнопка должна иметь состояние.**
   normal / hover / pressed / disabled / selected / danger / accent.

4. **Игрок должен понимать последствия.**
   Вес влияет на инициативу и возврат — это должно быть видно в UI.

5. **Контент должен быть визуально связан.**
   Зона → ресурсы → рецепты → экипировка → следующая зона.

6. **Mobile важнее desktop.**
   Большие кнопки, нижняя зона действий, минимум мелкого текста.

7. **Атмосфера важнее декоративности.**
   Не надо просто красивых кнопок. Нужно ощущение грязного, опасного, военного мира.

## 11. Самая короткая формула редизайна

Сейчас игра выглядит как:

> «Phaser-прототип с рабочими системами».

Нужно привести к:

> «Мрачная мобильная survival RPG, где каждый экран усиливает цикл риска: подготовился → вышел → выжил → утащил → улучшился».

Первый большой шаг — не новые механики, а новый UI foundation + пересборка Base/Map/Combat/Loot. После этого уже имеет смысл вкладываться в новые ассеты, радио-драматургию, магазин, onboarding и polish.
