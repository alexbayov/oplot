# Kickoff: Engineer — Веха M11.4 (Passive Skill Tree)

Ты — **Engineer** на M11.4 (Skill Tree — пассивная часть) проекта «Оплот».

Repo: https://github.com/alexbayov/oplot
Brief: `docs/redesign/m11/M11.4-skill-tree.md`
Base branch: `m12-integration`

## When

Start immediately. Параллельно с M11.1 (tiers), M11.5 (docs sync), M12.0 (combat foundation). Никаких прямых блокеров.

## Context

- Сейчас в игре **плоские 8 перков** (см. `content/perks.json`). Игрок получает 1 перк на уровень, выбирает из 3 случайных.
- M11.4 заменяет это на **дерево из 24 узлов (3 ветки × 8 узлов)**.
- В рамках MAX-варианта M12 split:
  - **M11.4 (эта веха):** 16 ПАССИВНЫХ узлов (8 в ветке Боец, 4 в Выживальщик, 4 в Ремесленник)
  - **M12.1 (позже):** 8 АКТИВНЫХ узлов как weapon-like abilities (в скоупе M12.1 active abilities)
- Игрок получает 1 очко на уровень (10 уровней → 10 очков → невозможно прокачать всё).
- Узлы внутри ветки открываются последовательно (узел 3 требует узел 2). См. топологию в спеке §3.

## Scope

1. **Перевести `content/perks.json`** на формат дерева:
   - Каждый узел: `{ id, name_ru, description_ru, branch, position, requires?: string, tier: 1..3, kind: "passive", effects: {...} }`
   - 16 пассивных узлов (см. таблицу в спеке §3.1, 3.2, 3.3)
   - Старые 8 перков мапятся в новые ID (миграция в state)
2. **`src/state/SkillTree.ts`** — новый модуль:
   - `getAvailableNodes(unlocked: string[]): Node[]` — какие можно открыть следующими
   - `canUnlock(nodeId, unlocked, level): boolean` — проверки prereq + skillPoints
   - `unlockNode(nodeId)` — атомарное добавление, mutates GameState.unlockedPerks
   - `getPassiveEffects(unlocked): PassiveEffectsBundle` — сумма всех эффектов (max_hp, regen, damage_mul, defense_mul, ...)
3. **Миграция сейвов:** старые `perks: string[]` (8 имён) → новые `unlockedPerks: string[]` (новые ID). Карта старое→новое:
   - `toughness` → `survivor_tough_1`
   - `quick_hands` → `fighter_quick_1`
   - `lucky_scavenger` → `crafter_lucky_1`
   - и т.д. (см. таблицу в спеке §6 Migration)
4. **UI:** новая сцена `SkillTreeScene.ts` — древо как 3 колонки × 8 узлов. Подсветка открытых (зелёные), доступных (жёлтые), заблокированных (серые). Клик на доступный → раскрывает modal «Открыть [Название] за 1 очко?». Кнопка «Открыть» вычитает 1 skillPoint, делает unlockNode, возвращает в сцену.
5. **Хук в `LevelUpScene`:** вместо рандомного выбора из 3 перков теперь просто +1 skillPoint и тост «Открой узел в дереве навыков».
6. **Применение passive эффектов:**
   - `max_hp` бонусы суммируются в `GameState.player.maxHp` при unlock
   - `damage_mul`, `defense_mul` читаются `combat.ts` (там уже есть perk multipliers, расширь)
   - `regen_per_turn` — новый поле, добавь в combat resolve
7. **Тесты (8 минимум):**
   - canUnlock prereq check (родительский узел не открыт)
   - canUnlock skillPoint check (нет очка)
   - unlockNode идемпотентен (повторно — no-op)
   - getPassiveEffects суммирует правильно (3 узла HP → +30 maxHp)
   - migration старого save с 8 перками → корректные новые ID
   - SkillTreeScene smoke-test (mock сцена)
   - LevelUpScene даёт +1 skillPoint
   - Combat читает damage_mul из дерева

## Anti-scope

- НЕ делать активные узлы (это M12.1)
- НЕ менять combat.ts кардинально — только подключи damage_mul/defense_mul/regen
- НЕ менять `applyAttack` сигнатуру (M12.0 это делает)
- НЕ трогать iconpack — иконки узлов реюзим существующие perk-иконки (8 есть, на 8 новых ставим placeholder TBD-M11.4-art)
