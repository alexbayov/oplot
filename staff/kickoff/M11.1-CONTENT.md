# Kickoff: Content Designer — Веха M11.1 (Tier System T1-T5)

Ты — **Content Designer** на M11.1 (Tier System) проекта «Оплот».

Repo: https://github.com/alexbayov/oplot
Brief: `docs/redesign/m11/M11.1-tier-system.md`
Base branch: `m12-integration` (она же содержит весь M11 work-in-progress)

## When

Start immediately. Параллельно с M11.4 (perks), M11.5 (docs sync) и M12.0 (combat). Никаких блокеров — `tier` field уже существует в items.json.

## Context

- M11.0a content PR залил 187 items. У каждого item уже есть **`tier: 1..5`** в JSON, но он расставлен в спешке (Devin Content) и НЕ сверен с реальной прогрессией зон/мобов.
- M11.1 — это **финальный балансировочный проход по тирам**. Без него M12 combat не сможет правильно соотносить «mob T3 vs weapon T3».
- Никаких новых items не добавляешь. Только редактируешь существующие.

## Scope

1. **Прочитать спеку:** `docs/redesign/m11/M11.1-tier-system.md` § «Привязка тиров к зонам / мобам / рецептам».
2. **Расставить tier на 187 items:**
   - Самоделы (8 шт): T1 нож, T1 заточка, T1 поджига, T2 трубочное, T2 копьё, T2 арбалет, T2 обрез, T3 молотов
   - Реальные стволы (22 шт): T2 ПМ/ТТ/Стечкин, T3 ППШ/СКС/АКС-74У, T3 АКМ/АК-74/Мосина/ИЖ-43, T4 РПК/СВД/Сайга-12/Бекас, T5 уникальные boss-drops
   - Партов (~60 шт): тир = тиру итогового ствола
   - Моды (8 шт): T2 базовые (ПБС-ПМ, прицел КС), T3 продвинутые (тактическая рукоять, расш.магазин), T4 уникальные
   - Боеприпасы (7 калибров): тир = тиру самого распространённого ствола под этот калибр
   - Resources (38 шт): T1 базовые (дерево/лом/ткань), T2 средние (электроника/порох/кожа), T3 редкие (микросхемы/спецсплав), T4 трофейные (зоновые токены)
   - Armor (15 шт): расставить по 5 тирам
   - Consumables (27 шт): расставить — простые бинты T1, медкиты T2, стимы T3
3. **Расставить tier на 11 mobs** (поле `tier` в `content/mobs.json`):
   - T1: wild_dog, pack_rat
   - T2: marauder, mutant
   - T3: looter_sniper, fanatic_berserker, armored_guard
   - T4: relic_drone, forest_alpha_mutant
   - T5: warehouse_drone_prime, city_guard_captain
4. **Расставить `min_player_level`** на 9 zone levels: T1 зоны — level 1, T2 — level 3, T3 — level 5, T4 — level 7, T5 — level 9 (см. таблицу в спеке).
5. **Расставить `tier` на 71 рецепт**: тир рецепта = тиру результата.
6. **Sanity-проверка инвариантов** (см. спеку §5):
   - НЕТ T5 предметов в T1 зонах
   - НЕТ T1 рецептов которые требуют T3+ ингредиенты
   - НЕТ мобов T3+ с дропом T5 (только боссы)

## Anti-scope

- НЕ добавлять новых items/recipes/mobs (это работа M12 content или M13)
- НЕ менять стат-числа (damage_min/max, hp, defense) — балансировка чисел в M11.5 docs sync проходит отдельно
- НЕ трогать `src/` код
- НЕ менять `tier` field type — это уже `ItemTier = 1|2|3|4|5` в `src/types/item.ts`
