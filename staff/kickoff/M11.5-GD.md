## When

Start immediately. Параллельно с M11.1 (tiers), M11.4 (perks), M12.0 (combat). Никаких блокеров — это работа с документами, не с кодом.

## Context

Текущие doc-файлы рассинхронизированы с реальностью кода после M11:
- **`docs/GDD.md` (2268 строк)** — описывает старую систему 80 items без модов/партов/калибров
- **`docs/balance.md`** — числа из M9 балансировки, без учёта 22 новых стволов
- **`docs/content-brief.md`** — план контента до M11.0a content drop
- **`docs/style-guide.md`** — без новых iconpack guidelines для парт/мод иконок

После M11 без docs sync новый разработчик/Devin **не сможет понять текущую модель**. Перед M12 это критически — Combat Pass опирается на актуальное понимание оружия.

## Scope

### 1. GDD.md — мажорный апдейт
- **§ Item Model:** перепиши под трёхклассовую систему (craft / drop / mod). Добавь поля: caliber, magazineSize, durability, parts, mods, fits_weapons, name_real_ru, name_generic_ru
- **§ Combat:** добавь упоминание что после M12 будут statuses, abilities, positions (короткая ссылка на M12-COMBAT.md, без дублирования)
- **§ Crafting:** обнови — теперь 2 режима (Крафт самоделов / Сборка реальных стволов). Добавь раздел про разбор broken_craft на верстаке (30% возврат)
- **§ Loot:** обнови формат — теперь `mob.drops` (новый) и `mob.drop_table` (legacy), bridge в loot.ts
- **§ Progression:** добавь Skill Tree (после M11.4) — 24 узла, 3 ветки. Перки старые помечены как deprecated
- **§ Tiers:** новая секция — система T1-T5, цвета рамок, привязка к зонам/мобам

### 2. balance.md — числовой ревью
- Просвери что M11.0a content числа стволов (damage_min/max, magazineSize, durability) соответствуют тирам
- Если найдешь дисбаланс (например T5 ствол слабее T3) — flag в `staff/handoff/M11.5-GD-rebalance-flags.md`
- Не правь числа сам — это работа M11.1 content или отдельной балансировочной вехи

### 3. content-brief.md — текущий статус
- Обнови раздел «что добавлено в M11.0a» — 107 новых items, 29 рецептов, drops на 11 мобах
- Зафиксируй список «что ещё нужно для M12» — иконки приёмов оружия (~30), иконки статусов (6), iconpack environment (бочка, машина, тело)
- Carve out для M13 (Точка + Зачистка) — содержательная часть после M12

### 4. style-guide.md — iconpack guidance
- Добавь раздел: иконки партов оружия — стилистика как существующие weapon icons, фон серый, размер 64×64
- Добавь раздел: иконки модов — золотой бордер, фон тёмно-зелёный (выделение от обычного оружия)
- Добавь раздел: иконки broken_craft — тот же ствол что оригинал, но с трещиной/перечёркнут красным

### 5. M11-CORE-LOOP.md — закрытие

### 6. ADR (Architecture Decision Record)
Создай `docs/decisions/ADR-001-three-class-items.md`:
- Контекст: почему пошли от «5 weapon_ranged» к «craft/drop/mod»
- Решение: гибридный формат legacy + M11 поля, ItemRegistry как фасад
- Альтернативы: чисто M11 переезд (отвергнут — break существующего)
- Последствия: 6 lint errors остаются как pre-existing, save migration требуется

## Anti-scope

- НЕ менять числа балансировки (только flag)
- НЕ писать новый контент (рецепты, items, мобы)
- НЕ трогать `src/` код
- НЕ переписывать GDD с нуля — додатачи и правки. Сохрани историю и стиль.
- НЕ создавать новых `docs/redesign/M*` файлов (это работа PM)
