# План разработки «Оплот»

> Цель: **выпустить survival RPG на Яндекс.Играх с замкнутым циклом «вылазка → бой → лут → крафт → база»**, потом наращивать контент, зоны и системы.

## 0. Концепция (зафиксировано Заказчиком)

- **Жанр:** Survival RPG с глубоким крафтом и пошаговым боем.
- **Сеттинг:** Постапокалипсис. Заброшенный город после техногенной катастрофы. Современное оружие, реалистичный стиль.
- **Core loop:** База → Вылазка → Бой (пошаговый) → Лут (вес!) → Возвращение → Крафт → Повторяй.
- **Уникальные механики:**
  - **Вес/жадность** — чем больше тащишь, тем медленнее и уязвимее.
  - **Радио/доверие** — ловишь сигналы, не знаешь правда или засада.
  - **Модульная сборка** — оружие и броня из конкретных компонентов конкретных зон.
  - **XP через бой** — Lineage-стиль, каждый моб = опыт + шанс дропа.
  - **Зоноспецифичные ресурсы** — мотивация исследовать разные зоны.
- **Визуальный стиль:** "Military Graphic Novel" — Darkest Dungeon meets Modern Military. Контрастные тени, bold линии, painted style.
- **Платформа:** Яндекс.Игры, HTML5, mobile-first.
- **Стек:** Phaser 3 + TypeScript + Vite + Yandex Games SDK.

## 1. Главное правило

> **Сначала «скелет»: одна зона, один бой, один крафт. Потом контент. Потом мета-системы. Потом красота.**

Каждая система — это отдельная веха. В MVP едет только вылазка + бой + лут + инвентарь. Всё остальное QA блокирует до прохождения предыдущей вехи.

## 2. MVP-Definition (что считается «играбельный скелет», M2)

Игрок может:
1. Открыть игру (загрузка < 5 сек на мобильном 4G).
2. Увидеть базу (экран Оплота) с кнопкой «Вылазка».
3. Выбрать зону «Лес» (единственная).
4. Встретить 1-3 мобов, провести пошаговый бой (нож + самодельный пистолет).
5. Собрать лут (3-5 ресурсов), видеть вес в рюкзаке.
6. Вернуться на базу, увидеть инвентарь.
7. Скрафтить 1-2 предмета из собранных ресурсов.

На MVP:
- [x] 1 зона (Лес Лв1)
- [x] 2 оружия (нож, самодельный пистолет)
- [x] 3 типа мобов (Мародёр, Дикий пёс, Мутант)
- [x] 5 базовых ресурсов (Дерево, Металлолом, Ткань, Еда, Вода)
- [x] 3-5 рецептов крафта (T1)
- [x] Система веса (влияет на скорость возврата)
- [ ] Нет радио
- [ ] Нет перков
- [ ] Нет боссов
- [ ] Нет модулей оружия
- [ ] Нет коммуны

## 4. Что НЕ делаем до релиза (Anti-scope)

- **Коммуна / NPC-члены** — пост-релиз, обновление #1.
- **Сюжетная кампания** — пост-релиз.
- **PvP / мультиплеер** — не делаем.
- **3D / изометрия** — не делаем.
- **Фэнтези-элементы** (магия, драконы) — запрещено.
- **Реальные ТМ** (АК-47, M4A1) — запрещено, используем обобщённые названия.

QA имеет право вето на любую попытку вставить эти фичи на любой вехе (anti-scope действует постоянно, не только до M9).

> **Прецедент (2026-05-29):** в корне репозитория обнаружен `aquarium.html` — 3D-аквариум на Three.js, нарушавший правило «нет 3D» и ломавший `npm run dev` (импорт неустановленного `three`). Удалён при чистке (ветка `chore/docs-sync-and-dev-fix`).

## 5. Definition of Done — общий для каждой вехи

Любая фича / веха считается готовой только если:
1. Есть описание в GDD (что и зачем).
2. Цифры есть в `balance.md` (если фича про числа).
3. Реализована Engineer-ом по GDD.
4. Контент написан Content Designer-ом по шаблону GDD (если фича про контент).
5. Прошла чек-лист QA для этой вехи.
6. Не сломала предыдущие вехи (smoke-тест).
7. Билд проходит, нет console.error, грузится в Chrome mobile-emulator.

## 6. Риски и митигейшены

| Риск | Митигейшен |
|---|---|
| Контент-провал (мало предметов → быстро приедается) | Чек-лист уникальности в content-brief. Цель: 80+ предметов к релизу |
| Балансовая поломка | `balance.md` как таблица. QA проверяет на каждой М-вехе |
| Перегруз механиками | QA с правом вето. Anti-scope |
