## Скоуп M4 (из PLAN §3 + PM kickoff decision)

«Перки и прогрессия». Игрок может:

1. Получать **XP** за убийство мобов (`mob.xp_reward`).
2. Накапливать XP → уровни по XP-curve (формула задаётся GD M4 amendment'ом).
3. На каждом level-up получать **выбор из 3 случайных** перков из пула 8 пассивных.
4. Просматривать список взятых перков + текущий уровень + прогресс до следующего на новой `ProgressionScene` (или вкладке `BaseScene`).
5. Видеть применённые модификаторы перков в бою / крафте / возврате (числовое влияние — задаёт GD).

**Пул из 8 пассивных перков (черновой, GD доточит на amendment'е):**

1. `tough_skin` — +HP_max
2. `sharp_blade` — +damage (melee + ranged)
3. `lean_pack` — -weight_penalty (return_time)
4. `lucky_scavenger` — +loot_quantity (drop quantity multiplier)
5. `keen_eye` — +crit_chance (доп. crit roll)
6. `reinforced_plates` — +armor_efficiency (mitigation multiplier)
7. `quick_hands` — +crafting_speed (craft_time multiplier)
8. `fast_learner` — +XP_gain

GD фиксирует точные числа в `balance.md` §M4.

## Anti-scope M4

- **Нет skill tree** (поинты + ноды + prereq'и) — это M5+ как «perk system refactor». В GDD M4 amendment явно зафиксировать «tree — evolution path для M5» (чтобы не было double-work).
- **Нет активных ability / cooldowns** — только пассивные multiplier'ы (active — M5+).
- Нет боссов / инстансов / T3 чертежей (M5).
- Нет полной радио-логики (M6).
- Нет Yandex SDK (M8).
- Нет реального арта / звуков / анимаций (M7).
- Нет UI-библиотек сторонних (Phaser built-in).
- Перки **сохраняются только в session memory** — real save через Yandex Cloud Saves под M8.
