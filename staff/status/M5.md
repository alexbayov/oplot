## Скоуп M5 (из PLAN §3 + PM kickoff decision)

«Боссы и инстансы». Игрок может:

1. Зачищать обычные мобов в зонах, **доходить до глубины 3** (forest=3, warehouse=3, city=3) — там встречает **босса** (`Mob.role = "boss"`).
2. Сражаться с боссом в **2-фазном бою**: при `HP < 50% × max_hp` происходит **phase transition** (smena поведения / новый damage modifier / визуальный шаг), бой продолжается до смерти босса.
3. Получать с босса **уникальный boss-drop** (1 гарантированный ресурс на босса) + повышенный XP-reward.
4. Использовать boss-drop как ингредиент для **T3 чертежа** (3 рецепта, 1/зона) — T3 экипировка с улучшенными статами.
5. Раз в сутки заходить на **дейли-инстанс** (`GameState.progress.daily_completed: { [zone_id]: timestamp }`) — повторный боссфайт за повышенный лут с cool-down 24ч.
6. В **газовых зонах** (`zone.depth_2..3` warehouse/city — флаг `is_gas: true`) получать **damage-per-turn** на возврате/в бою, если в инвентаре нет `gas_mask`.

## Anti-scope M5 (явный)

- **Нет модульного оружия / брони (head-slot, accessory, runes)** — это **M5+ отдельная подсистема** (refactor path вне M5). M5 = простой T3 craft, no equipment modules.
- **Нет полной радио-логики** — это **M6** (rewards / ambush / trust system). M5 RadioScene остаётся UI-stub (dismiss-only).
- **Нет Yandex SDK / Cloud Saves / Leaderboard / IAP** — это **M8**.
- **Нет skill tree / поинтов / prereq / tier / cost / cooldown** — это **M5+ refactor path** (не делается в M5 ни даже PvP).
- **Нет PvP / мультиплеера / community** — пост-релиз / не делается.
- **Нет boss-cinematics / animated phase transition** — это **M7 polish**. M5 phase transition = текстовый popup / static sprite swap.
- **Нет дейли-instance reward rotation / weekly events** — M5 daily = простой 24h cool-down + повторный boss drop, без вариативности.
- **Нет дополнительных AI behaviors** — M5 boss AI **переиспользует M3 5 behaviors** (`ranged_keep_distance` / `defensive_cover` / `berserker_low_hp` / `pack_bonus_when_paired` / `armor_piercing_ranged`) + phase swap.

QA Spec / Acceptance имеют право vetoить любую попытку расширить scope до этих фич.
