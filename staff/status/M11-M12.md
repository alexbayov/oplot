## QA acceptance checklist — M11/M12 product gate

QA owns the PASS/REWORK call. Release announcement stays blocked until all critical checks pass. Record evidence for each manual item: device/browser, save fixture, observed result, screenshots/video if UI-related, and console status.

### A. Automated gates

- [ ] `npm run typecheck` passes.
- [ ] `npm run lint` passes.
- [ ] `npm run test` passes.
- [ ] `npm run build` passes.

### B. Save migration v1/v2 → v3

- [ ] Load a **legacy v1-style save** (snapshot before explicit `version` field, if fixture exists) and confirm migration finishes without crash.
- [ ] Load a **v2 save** and confirm it migrates to v3 without losing player HP, XP/level, perks, inventory, base stash, radio trust, settings, and sortie state where applicable.
- [ ] Load an **old save without `version`** and confirm it is treated as legacy input, migrated safely, and saved back with the current version.
- [ ] Confirm migrated inventory items resolve through ItemRegistry, including M11 item classes/tier fields.
- [ ] Confirm migration failure path does not corrupt the previous cloud/local data; QA can reload/retry from the original fixture.

### C. Combat engine smoke

- [ ] **Ordinary fight:** enter a normal non-boss combat from sortie; player/enemy turns advance; no soft-locks.
- [ ] **Victory:** kill enemy, receive expected post-combat transition, loot opportunity, XP/progression effects, and no console.error.
- [ ] **Defeat:** let player lose; defeat/second-chance/return handling is stable and does not corrupt save.
- [ ] **Escape / return flow:** if escape is available in current UI, use it and verify correct return/penalty flow; if not available, verify normal ReturnScene path after combat/loot.
- [ ] **Durability:** weapon durability changes when expected; broken/low-durability behavior is visible and does not crash combat or inventory.
- [ ] **Ammo / caliber:** if current weapon/ammo flow is active, verify correct caliber is consumed/required and wrong/missing ammo fails gracefully.
- [ ] **Modifiers / effects:** verify at least one active modifier/status/effect applies, ticks or expires, renders in UI, and does not block turn progression.
- [ ] **Enemy abilities/phases:** verify signature/phase behavior for at least one relevant mob/boss and confirm legacy mobs still work.

### D. Skill tree

- [ ] Open skill tree/progression UI from a clean/new save without crash.
- [ ] Unlock a valid skill node and verify cost is paid and effect/state is visible.
- [ ] Attempt to unlock a node with unmet prerequisites; verify it is blocked with readable feedback and no state mutation.
- [ ] Save/reload after unlock; verify skill state persists locally/cloud where applicable.
- [ ] Reset behavior: if reset exists, verify reset cost/result; if reset does not exist, verify there is no misleading reset UI or accidental reset path.
- [ ] Verify skill effects used by combat/progression do not break existing M0–M10 flows.

### E. Item tiers T1–T5

- [ ] Tier badges/text for T1, T2, T3, T4, and T5 items display correctly where those items are available.
- [ ] Crafting with tiered items works for at least one valid recipe and fails gracefully for missing ingredients.
- [ ] Loot can grant tiered items without broken IDs or blank names/icons.
- [ ] Inventory shows tiered items, generic release-safe names, stack/count/durability where applicable, and no real weapon TM leaks in runtime display.
- [ ] Weight calculation includes tiered items correctly; overweight/return risk UI remains understandable.
- [ ] Moving tiered items between backpack/base stash preserves IDs, counts, durability/mod data where applicable.

### F. Mobile / landscape UX

- [ ] Game renders at **1280×720 landscape** using Scale.FIT without cropped critical UI.
- [ ] Touch targets for combat actions, inventory/craft buttons, skill nodes, map/base hotspots, ads/IAP buttons are usable on mobile viewport.
- [ ] Combat text, status labels, tier badges, item names, weight readout, and skill descriptions are readable on mobile landscape.
- [ ] Orientation/resize does not hide required controls or trap the player.
- [ ] No blocker UX issue is seen in the first 10 minutes of a new-player smoke run.

### G. Yandex release risks

- [ ] **Cloud save:** load/save works on Yandex Draft or fails soft with local/offline fallback and no data loss.
- [ ] **Rewarded ads:** reward grants only after rewarded callback; unavailable/error paths fail soft and do not block progression.
- [ ] **Interstitial:** post-sortie/interstitial placement does not interrupt critical state transitions and fails soft.
- [ ] **IAP:** catalog/purchase/unprocessed-purchase flows work on Draft or are explicitly marked blocked; failed purchases do not grant rewards.
- [ ] **Offline/local fallback:** game remains playable when YaGames SDK, ads, IAP, or cloud save are unavailable.
- [ ] No release-blocking `console.error` appears during Draft smoke.

### H. Regression — M0–M10 core loop

- [ ] Boot → BaseScene loads successfully.
- [ ] Base → Map → Sortie starts correctly.
- [ ] Sortie → Combat works with current content.
- [ ] Combat → Loot → Return resolves without state loss.
- [ ] Return merges backpack loot to base stash and restores/updates expected player state.
- [ ] Crafting still works after return.
- [ ] Radio/encounter/progression/base hotspots still open or fail soft according to existing M10 behavior.
- [ ] Existing tests/manual smoke do not reveal regressions in M0–M10 scenes.

## Definition of Done — M11/M12 acceptance

M11/M12 become product-accepted only when all of the following are true:

- [ ] All automated checks are green: `typecheck`, `lint`, `test`, `build`.
- [ ] Manual checklist sections B–H are passed or each non-applicable item is explicitly marked N/A with reason.
- [ ] No `console.error` appears in the smoke run.
- [ ] No save corruption is observed across migration, save/reload, cloud/local fallback, and post-combat persistence.
- [ ] No blocker UX issue appears in the first 10 minutes of mobile landscape gameplay.
- [ ] QA posts final verdict: **PASS** for release announcement, or **REWORK** with blockers.
