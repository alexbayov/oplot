# Команда проекта «Оплот»

> Кодовое имя проекта: **Oplot**
> Платформа: Яндекс.Игры (HTML5, mobile-first)
> Заказчик / Визионер: **dev07** (Alex)
> Продюсер / Дирижёр: **Devin (PM-сессия)**

## Принцип организации

Это **не индивидуальная разработка**. PM **не пишет код руками** — раздаёт задачи на роли и следит за интеграцией. Каждая роль имеет:
- **Цель** (зачем существует)
- **Артефакты** (конкретные файлы на выходе)
- **Что НЕ делает** (анти-зона ответственности)
- **Триггер** (когда вступает в работу)

Если роль не создала файл по своему стандарту — задача не считается принятой.

## Состав команды (6 ролей)

| Роль | Кто играет | Главный артефакт | Зона ответственности |
|---|---|---|---|
| **Продюсер (PM)** | Devin (текущая сессия) | `staff/PLAN.md`, бэклог, ревью | Координация, приоритеты, общение с заказчиком, нарезка задач |
| **Game Designer** | Devin (child-сессия) | `docs/GDD.md`, `docs/balance.md` | Игровые системы, формулы, баланс, описания механик ДО кода |
| **Content Designer** | Devin (child-сессия) | `content/*.json` | Предметы, мобы, рецепты, зоны, радио-сигналы, дроп-таблицы |
| **Engineer** | Devin (child-сессия) | `src/` (Phaser 3 + TS) | Реализация по GDD, UI, SDK, оптимизация |
| **Artist / Asset Lead** | Devin (child-сессия) | `assets/`, `docs/style-guide.md` | AI-генерация спрайтов, UI-кит, атласы, стиль |
| **QA / Критик** | Devin (child-сессия, ВСЕГДА отдельная!) | Чек-листы, тест-репорты | Definition of Done, anti-scope-крип, вето, тесты |

Минимально жизнеспособная команда — **6 ролей**. Артефакты **всегда** разделены.

## Кадровая модель

| Роль | Кто физически | Как технически |
|---|---|---|
| **Заказчик / Прод** | dev07 (Alex) | Решения «делаем / не делаем», апрув вех, блокирующие вопросы |
| **PM** | Devin-сессия (PM) | Промты, сбор артефактов, контроль сроков, обновление PLAN.md |
| **Game Designer** | Devin child-сессия | Пишет GDD.md, balance.md |
| **Engineer** | Devin child-сессия | Код, билд, SDK-интеграция |
| **Content Designer** | Devin child-сессия | items.json, mobs.json, recipes.json, zones.json, radio.json |
| **Artist** | Devin child-сессия + AI-генерация | Спрайты, фоны, UI, style-guide |
| **QA** | Devin child-сессия (всегда отдельная от фичи!) | Чек-листы, ревью, вето |

**Ключевой принцип:** QA всегда в отдельной сессии от той, что делала фичу.

## Цепочка передачи задач (workflow)

```
Заказчик (dev07)
   ↓ хочу фичу X
PM (Devin)
   ↓ декомпозирует и распределяет
   ├→ Game Designer:  пишет спеку в GDD (что и зачем)
   ├→ Content Designer: пишет контент (JSON)
   ├→ Artist:          генерирует ассеты
   ↓
Engineer: получает GDD + контент + ассеты → пишет код
   ↓
QA: ревью по чек-листу → блочит или пропускает
   ↓
PM: мерджит, закрывает веху
```

## Право вето

| Роль | Может заблокировать |
|---|---|
| QA / Критик | Любую новую фичу до закрытия MVP. Любой релиз с непройденным чек-листом. |
| Game Designer | Реализацию механики, отличающейся от GDD. |
| PM | Любую активность не из плана. |
| Заказчик | Всё. |

## Артефакты-источники правды

| Тип знания | Где живёт | Кто пишет |
|---|---|---|
| Что мы делаем и зачем | `staff/PLAN.md` | PM |
| Как работает игра | `docs/GDD.md` | Game Designer |
| Баланс и формулы | `docs/balance.md` | Game Designer |
| Контент (предметы, мобы, рецепты) | `content/*.json` | Content Designer |
| Стиль и ассеты | `docs/style-guide.md`, `assets/` | Artist |
| Правила приёмки | Чек-листы QA | QA |
| Код | `src/` | Engineer |
| Описания ролей | `staff/roles/*.md` | PM |

Если фича есть в коде, но её нет в GDD — **это баг процесса**, QA должен поймать.

## Что НЕ роль команды (out-of-scope)

- Маркетинг и продвижение
- Многопользовательский режим
- Бэкенд (всё клиентское, сейвы через Yandex SDK)
- Локализация на иностранные языки (только RU на старте)
- Фэнтези-элементы
