## API для следующих сессий

### Для Content Designer (M11.0c)

После того как напишешь новый `items.json` с M11Item-shape, в BootScene добавь:

```ts
import { setM11Items } from "../state/ItemRegistry";
// ...после parseContent:
setM11Items(parsedItemsArray);
```

До этого ItemRegistry автоматически fallback'ится на legacy items через `adaptLegacyItem`. Старый items.json продолжает работать без изменений.

### Для UI (M11.0d)

```ts
import { getItem, itemName, itemNameById } from "../state/ItemRegistry";

const pm = getItem("pistol_t2_pm");        // M11Item | null
const name = itemName(pm);                  // учитывает WEAPON_NAMING_MODE
const nameById = itemNameById("pistol_t2_pm"); // shortcut
```

### Для боя/инвентаря (M11.0d+)

```ts
import { createWeaponInstance, installMod, removeMod, computeWeaponStats } from "../state/ItemRegistry";
import { onCraftWeaponBreak, onDropWeaponBreak, damageWeapon } from "../systems/durability";

// Сборка ствола из частей:
const inst = createWeaponInstance("pistol_t2_pm", parts);

// Установка мода:
const { instance, replaced } = installMod(inst, "mod_suppressor_9x18");

// Снятие (с шансом поломки):
const { instance, modSurvived, modId } = removeMod(inst, "muzzle");

// Итоговые стэты с модами:
const stats = computeWeaponStats(inst);

// Урон по экземпляру:
const { instance: damaged, broken } = damageWeapon(inst, 5);
if (broken) {
  const { inventory, returnedParts } = onDropWeaponBreak(damaged, currentInventory);
}
```

## Открытые вопросы для следующего PR (M11.0b — Mods crafting)

1. **Mod removal break chance** — сейчас в каждом моде поле `removalBreakChance: 0.2`. Возможно стоит сделать его глобальной константой в `balance.ts` и читать оттуда (как сейчас сделано с `MARAUDER_FLEE_HP_RATIO`).
2. **WeaponInstance в инвентаре** — текущий `InventoryStack` хранит `{ item_id, count }`. Для WeaponInstance нужен отдельный массив в `PlayerState`. Это **сделает M11.0d Engineer** при работе с UI инвентаря.
3. **Legacy adaptation для armor** — пока armor → "resource". Когда контент-сессия даст armor M11-shape, адаптер нужно расширить (или просто убрать armor из адаптации).
