## Твоя задача

Проверить GDD.md и balance.md на:

### Чек-лист ревью

#### Полнота (всё ли описано для MVP)
- [ ] Описана вылазка (выбор зоны → бой → лут → возврат)
- [ ] Описан бой (пошаговый, 2 типа оружия, 3 действия)
- [ ] Описаны 3 моба с уникальными статами и поведением
- [ ] Описан инвентарь и система веса
- [ ] Описан крафт (5 рецептов)
- [ ] Описаны JSON-схемы (Item, Mob, Recipe, Zone)

#### Баланс (числа адекватны)
- [ ] Герой выживает 1-2 боя без аптечки
- [ ] Мародёр — слабый (убивается за 3-4 удара ножом)
- [ ] Мутант — сильный (нужен пистолет или аптечка)
- [ ] Вес реалистичен (30 кг рюкзак, нож ~0.5 кг, пистолет ~1.5 кг)
- [ ] XP-прогрессия: уровень 2 за ~5-8 боёв
- [ ] Рецепты логичны (ресурсы соответствуют результату)

#### Anti-scope (нет ли лишнего)
- [ ] НЕТ радио-системы
- [ ] НЕТ перков/навыков
- [ ] НЕТ модульной брони
- [ ] НЕТ мини-боссов
- [ ] НЕТ больше 1 зоны
- [ ] НЕТ Yandex SDK

#### Непротиворечивость
- [ ] Числа в GDD совпадают с balance.md
- [ ] JSON-схемы покрывают все упомянутые поля
- [ ] Формулы используют поля из JSON-схем

## Результат

Отправь заказчику **блокирующим** сообщением ОДИН из:

### Если всё ок:
```
✅ QA Spec Review M1: APPROVE
Все чек-поинты пройдены. Можно запускать Engineer, Content, Artist.
```

### Если есть проблемы:
```
⚠️ QA Spec Review M1: CHANGES_REQUESTED

Проблемы:
1. [Конкретная проблема + что исправить]
2. [Конкретная проблема + что исправить]

Остальное ок.
```

- ❌ НЕ одобряй если числа отсутствуют (только описания без формул)
