## API/Schema notes для Engineer

### Item Model — добавленные поля (opt-in через ItemRegistry §1)

```ts
interface Item {
  // existing
  id: string;
  name_ru: string;
  type: "weapon_melee" | "weapon_ranged" | "weapon_part" | "modification" | "ammo" | "armor" | "consumable" | "resource";
  tier: 1 | 2 | 3 | 4 | 5;
  weight_kg: number;
  description_ru: string;
  flavor_ru: string;
  recipe_id: string | null;
  stats: Record<string, unknown>;

  // M11.0a additions (opt-in)
  item_class?: "craft" | "drop" | "part" | "modification" | "ammo" | "armor" | "consumable" | "resource" | "misc" | "broken_craft";
  name_real_ru?: string;     // "АК-74"
  name_generic_ru?: string;  // "Автомат 5.45×39"
  caliber?: Caliber;          // weapon_ranged + ammo
  durability_max?: number;    // craft weapons + drop weapons (M11.0b/c usage)
  mod_slots?: number;         // drop weapons
  parts?: string[];           // drop weapons → сборный из этих partов
  mod_slot?: string;          // modification items → "muzzle" | "optic" | "mag" | "grip"
  fits_weapons?: string[];    // modification items → ids стволов
  _migrated_from?: string;    // diagnostic: old id перед миграцией (опционально удалить после саве v2 migration)
}
```

### Recipe Schema — добавленные поля

```ts
interface Recipe {
  // existing fields
  id: string;
  result_id: string;
  result_count: number;
  ingredients: { item_id: string; count: number }[];
  tier: number;
  unlock_condition: string | null;
  craft_time_s: number;

  // M11.0a additions
  recipe_type?: "craft" | "assemble";  // "assemble" = сборка ствола из партов
  is_mod?: boolean;                     // для UI: показывать ли в табе «Моды»
}
```

### Mob Schema — добавленные поля

```ts
interface Mob {
  // existing
  id: string; name_ru: string; ...

  // M11.0a additions
  drops?: { id: string; chance: number; count?: [number, number] }[];
}
```

### Drop semantics

- `chance: 0.8` = 80% шанс что предмет выпадет (независимо от других)
- `count: [1, 3]` = если выпало, случайно от 1 до 3 штук (uniform). Если отсутствует — 1.
- `count: undefined` = всегда 1
- Несколько drops могут сработать одновременно (independent rolls)
- Спека §7.4 разделяет `guaranteed_drops` (chance:1.0) и `random_drops` — в JSON всё нормализовано к одному формату с `chance:1.0`

## Edge cases & decisions

### `knife` vs `shiv`
Spec §11.2 явно говорит: оба сохраняются как отдельные `craft_knife`, `craft_shiv` (разные характеристики). Реализовано: оба теперь в items.json как craft предметы с разной durability и damage.

### Стволы без T2 partов
АПС (T3), ППШ (T3) — добавлены источники на `city_guard_captain`, `fanatic_berserker` (T2-T3 мобы). Это создаёт пересечение T2/T3 части — engineer должен помнить что **tier_of_part = tier_of_weapon** не строго совпадает с tier-распределением мобов. Это OK по спеке.

### Drop-stubs (резервные)
11 stub-предметов добавлены за пределами явного спека для cross-reference валидации:
- `microchip`, `rubber`, `drone_core`, `leather_jacket`, `armored_vest_part`, `alpha_pelt`
- 3 чертежа: `armored_vest_recipe`, `rare_meds_recipe`, `unique_svd_blueprint`
- 2 базовых ресурса: `raw_meat`, `bone` (использовались в drops но не были в items.json)

Все они помечены как `resource` с placeholder descriptions. Engineer в M11.0c добавит реальную семантику.

### Recipe IDs
- Новые: `recipe_craft_shiv`, `recipe_pm`, `recipe_assemble_pm`, `recipe_mod_pbs1` (по convention `recipe_<itemId>` или `recipe_assemble_<weaponId>`)
- Мигрированные: старые `recipe_pipe_rifle` → `recipe_craft_pipe_rifle` и т.д.

### M-2 follow-up для CONTENT/Engineer
~5 weapon-like существующих items (`prime_shotgun`, `cleaver`, `circuitry` (это resource на самом деле), и т.д.) НЕ покрыты миграцией т.к. не пересекаются с новыми IDs. Они остаются как есть, помечены `item_class` по типу. PM может решить deprecated на этапе M11.5 Docs Sync.
